Method Deep Divelearned topology vs execution protocol

两种“结构化协作”：学习通信图 vs 工程化共享状态

AgentConductor

trained orchestrator · competition code
Actionπθ 每个 turn 输出 variable-length YAML DAG：roles、edges、execution order。
Learning先 SFT 学会合法 topology，再用 GRPO 联合优化正确率、有效性与 density。
Online adaptation执行 G(k)，读入 roles/code feedback z(k)，下一 turn 生成 G(k+1)。
EvaluationAPPS、LiveCodeBench v4、CodeContests、HumanEval、MBPP；pass@1。

CAID

runtime protocol · shared repository
Actionmanager 构建 dependency DAG，仅从 ready-set 向工程 Agent 分派任务。
Isolation每个 engineer 独立 git worktree；禁止靠 prompt “软隔离”共享目录。
Event loop任一 engineer 完成 → self-test → commit → merge → 更新依赖 → 再分派。
EvaluationPaperBench 与 Commit0-Lite；同一底座模型，对比 single-agent。

Multi-LLMSecCodeEval heterogeneous + tool

P6–P9（LLM-only）：SecLLMEval 最多 +22.33 pp；SecLLMHolmes 只有 −1.42 至 +0.48 pp。
P10（hybrid）：97.32% / 99.06% Secure Code Rate；组合 ensemble、CodeQL、cross-model verification。

注意：Secure% 受 CodeQL v2.11.2 覆盖范围定义；论文自己的 ablation 也显示换 Bandit 会产生大量 false negatives。

Teams Hold Experts Back negative result

5 个 ML benchmarks 上，team 距离 per-problem oracle expert 的 relative synergy gap 为 6.3%–41.1%；即使 reveal expert 也未消失。

Setting：异构 frontier models、4 rounds free discussion、最终 majority vote。结论针对 self-organizing deliberation，而非所有结构化 MAS。

Compute matching. CAID 的质量提升伴随更高成本；不能直接解释为“并行更快”。
Evaluator dependence. Secure-code 结果高度依赖静态分析器能检测哪些 CWE。
Scope. AgentConductor 测算法题，CAID 测长程仓库；两类结论不能互换。

结论层级我们现在能说什么
较强证据长程共享仓库中，物理隔离 + merge/test protocol 比 prompt 软隔离可靠。
较强证据安全代码中，异构模型只有接上高精度 analyzer，收益才稳定且显著。
负面证据无结构的异构团队不能假设达到 best-member performance；共识会稀释专家。
尚未回答在严格 compute-matched、同一 benchmark 下，learned topology、runtime routing、single strong agent 谁最好？
尚未回答coding 场景能否定义 per-instance expert oracle，并直接测 strong synergy，而非只报绝对 pass@1？
建议统一报告： Task score + best-member baseline + oracle upper bound + total tokens + model/API cost + test calls + wall-clock。
