PackLLM (Mavromatis et al. 2024)

M038arXiv 2024verified source ↗
PackLLM ('Pack of LLMs') fuses arbitrary user-specified LLMs at test time by weighting each model according to how well it minimizes perplexity on the input prompt, with no trained fusion module.

Background

Combining knowledge from multiple LLMs can improve task performance, but existing fusers are limited.

Problem

Learning-based fusers do not generalize to newly added models, and other test-time methods ignore how well each LLM actually understands a given input.

Motivation / Analysis

Perplexity of an LLM on the input prompt reflects its expertise for that input, providing a training-free signal for weighting models per query.

Key Insight

Solving a per-prompt optimization that minimizes ensemble perplexity yields importance weights that reflect each model's expertise, enabling plug-and-play fusion of arbitrary models.

Challenges

Solving the perplexity-minimization weighting efficiently at inference, and supporting arbitrary heterogeneous model pools without retraining.

Method

Two variants: PackLLM-sim uses perplexity directly as an expertise indicator, and PackLLM-opt approximately solves the perplexity-minimization problem via a greedy algorithm to set per-model importance weights for logit-level fusion at inference time.

Experimental Setup

Conducted with over 100 total LLMs across a diverse set of tasks, compared against test-time fusion baselines and learning-based fusion approaches.

Results

PackLLM outperforms test-time fusion baselines by 1.89% accuracy points and improves over learning-based fusion by 3.92-11.94% when leveraging new LLMs (confirmed in abstract).

Limitations

Requires access to model logits/perplexity (not black-box APIs), and greedy optimization is an approximation to the true perplexity-minimizing weights.

Relation to the Field

A test-time logit-fusion method in the same family as DeePEn (M037) and FuseLLM (M039); its per-input weighting is a lightweight alternative to LLM-Blender's trained ranker (M009); an 'during-inference' method per the ensemble survey (M041).
