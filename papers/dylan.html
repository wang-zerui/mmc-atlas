DyLAN (Liu et al. 2023)

M053COLM 2024pending audit source ↗
DyLAN is a Dynamic LLM-Powered Agent Network that selects a task-specific team using an unsupervised Agent Importance Score and lets agents collaborate dynamically, deactivating weak contributors.

Background

Multi-agent LLM collaboration usually fixes the number of agents and uses static communication structures regardless of the query.

Problem

Fixed teams and static structures waste computation and include unhelpful agents, hurting task-oriented collaboration.

Motivation / Analysis

Dynamically selecting and pruning agents per task could raise accuracy while keeping computational cost moderate.

Key Insight

An unsupervised Agent Importance Score can rank agent contributions without labels, enabling team optimization, while a temporal feed-forward-network view of the network supports dynamic, query-dependent collaboration.

Challenges

Scoring agent contribution without ground truth and deciding mid-run which agents to keep active.

Method

Two stages: (1) Team Optimization selects the best-contributing agents via the Agent Importance Score; (2) Task Solving lets the selected agents collaborate dynamically based on the query.

Experimental Setup

Evaluated on code generation, decision-making, general reasoning, and arithmetic reasoning tasks against strong single-agent and multi-agent baselines.

Results

Reports up to +25.0% accuracy on specific MMLU subjects via team selection; abstract also claims outperforming baselines across code generation, decision-making, reasoning, and arithmetic at moderate cost. Cited +13.0% MATH and +13.3% HumanEval over single GPT-3.5-turbo are from secondary summaries and not primary-audited here.

Limitations

Reported magnitudes (MATH/HumanEval) come from summaries and were not independently verified; the Agent Importance Score is heuristic and unsupervised.

Relation to the Field

An early dynamic team-selection method; foreshadows later reward-guided topology evolution (HERA) and diversity-aware scaling analyses.
