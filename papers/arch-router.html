Arch-Router (Tran et al. 2025)

M033arXiv 2025verified source ↗
A compact 1.5B router maps queries to user-defined domain/action preferences, decoupling routing policy from model assignment so new models can be added without retraining.

Background

LLM routing must choose among models with different strengths, styles and cost/latency, but benchmarks miss subjective human preferences and often use only a small model pool.

Problem

Existing routers optimize benchmark accuracy that ignores subjective preference, and adding or swapping models typically requires retraining the router.

Motivation / Analysis

Real deployments want routing that reflects human/operator preferences (which model should handle travel queries, image editing, etc.) and that is easy to reconfigure as the model roster changes.

Key Insight

Separating the route policy (query to domain/action preference) from the model mapping (preference to concrete model) lets operators encode preferences and remap models freely without touching the learned router.

Challenges

Learning to reliably map free-form queries to user-defined domains and action types with a small, low-latency model while matching human preference judgments.

Method

Arch-Router is a compact 1.5B model trained to map queries to user-defined domains (e.g. travel) or action types (e.g. image editing); a separate configurable table maps those preferences to models, so new models slot in without retraining or architecture changes.

Experimental Setup

Evaluated on conversational datasets against top proprietary LLMs on preference-matching. Model is released on Hugging Face (katanemo/Arch-Router-1.5B).

Results

Outperforms top proprietary LLMs on preference matching by 7.71% on average, reaching 93.17% accuracy at 51ms latency (verified in claim_checked).

Limitations

Requires users to define domains/actions; preference-matching evaluation is on conversational data and may not capture raw task accuracy.

Relation to the Field

Shares GraphRouter's no-retraining-for-new-models goal but achieves it via preference decoupling rather than graph induction; a practical, deployment-oriented router in this family.
