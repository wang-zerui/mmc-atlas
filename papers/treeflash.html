TreeFlash (Rheinboldt et al. 2026)

M195arXiv 2026verified source ↗
Adds a lightweight MLP conditioned on the drafter's hidden state and previous token to approximate the autoregressive distribution for one-shot block/tree drafters at O(1) decode cost, correcting parallel-drafting drift.

Background

One-shot block drafters generate an entire draft in a single forward pass for high throughput, but condition each token only on prefix context, ignoring previously drafted tokens (non-autoregressive conditioning).

Problem

Reduce the distribution drift of one-shot/parallel drafters relative to the verifier's true autoregressive distribution without giving up O(1) decoding.

Motivation / Analysis

Non-autoregressive conditioning causes the drafter's distribution to drift from the verifier's as draft depth grows, and it is worse in tree drafting where branches share the same marginal distribution.

Key Insight

A small MLP conditioned on the drafter hidden state plus the previous token can approximate the autoregressive distribution while preserving the constant-time decoding of one-shot drafters.

Challenges

Approximating autoregressive conditioning cheaply; keeping O(1) decode complexity; correcting shared-marginal drift across tree branches.

Method

TreeFlash adds an MLP layer conditioned on the drafter's hidden state and the previous token to approximate an autoregressive distribution, using a two-stage approximation mechanism that preserves O(1) decoding time complexity.

Experimental Setup

Evaluated across a variety of tasks and models against marginal tree drafting (per abstract).

Results

+12% block efficiency and +9% speedup over marginal tree drafting (verified in claim_checked), reported as state-of-the-art across tasks/models. The abstract describes it as approximate AR with no distribution-preservation claim, so it is approximate/lossy (not proven lossless).

Limitations

Only approximates the autoregressive distribution; no losslessness guarantee; adds a small MLP that must be trained; arXiv preprint.

Relation to the Field

Addresses the same marginal-drift problem in parallel/tree drafting that BASTION tackles via budget-aware tree search; a drafter-side refinement for block/tree speculative decoding.
