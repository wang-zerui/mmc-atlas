Lookahead Decoding (Fu et al. 2024)

M080ICML 2024verified source ↗
A draft-model-free, exact parallel decoding algorithm that uses Jacobi iteration and n-gram trajectories to break the sequential dependency of autoregressive decoding.

Background

Standard autoregressive LLM decoding is memory-bandwidth-bound, giving high latency and underusing accelerator parallelism; speculative decoding needs a draft model that is nontrivial to obtain and generalize.

Problem

Accelerate LLM decoding losslessly without requiring any auxiliary draft model or datastore.

Motivation / Analysis

Trading extra per-step compute for fewer total decoding steps is attractive when decoding is memory-bound and FLOPs are underused.

Key Insight

Casting decoding as a Jacobi fixed-point iteration produces n-gram trajectories that can be generated and verified in parallel, yielding exact output while reducing the number of sequential steps.

Challenges

Making parallel Jacobi trajectories converge efficiently to the exact autoregressive output; keeping it compatible with memory-efficient attention.

Method

Lookahead decoding runs parallel Jacobi iterations to guess and refine future tokens as n-grams, verifies candidates against the model, and commits verified prefixes; it uses no draft model and is compatible with FlashAttention.

Experimental Setup

Evaluated on MT-Bench and code completion tasks, including multi-GPU strong-scaling experiments.

Results

Up to 1.8x on MT-Bench and up to 4x with strong scaling across multiple GPUs on code (verified in claim_checked). It is an exact parallel decoding algorithm and thus distribution-preserving (lossless), needing no draft model.

Limitations

Trades increased per-step FLOPs (log-scale) for fewer steps, so gains depend on hardware having spare compute; single-GPU speedups are more modest than multi-GPU.

Relation to the Field

A draft-model-free counterpart to speculative decoding; like Self-Speculative Decoding and REST it avoids a separate drafter, but via Jacobi/n-gram parallelism rather than layer-skipping or retrieval.
