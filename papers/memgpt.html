MemGPT (Packer et al. 2023)

M098arXiv 2023 (later COLM 2024); now the Letta frameworkverified source ↗
MemGPT applies operating-system memory-hierarchy ideas to LLMs, letting the model self-manage tiered memory - a fixed main context (like RAM) plus external storage (like disk) - via function calls, so it behaves as if it had a far larger context window.

Background

LLMs have fixed context windows that cap how much conversation history or document text they can attend to at once.

Problem

This bounded context makes LLMs unable to sustain long conversations or reason over documents that exceed the window, limiting their usefulness for long-horizon tasks.

Motivation / Analysis

Traditional operating systems already solve the problem of limited fast memory via virtual memory and paging between tiers, suggesting an analogous solution for LLM context.

Key Insight

The LLM itself can act as its own memory manager: given tools to page data between main context and external storage and interrupts to manage control flow, it can decide what to keep, evict, and retrieve to create the illusion of unbounded context.

Challenges

Teaching the model when to page information in/out, summarizing recursively without losing key facts, and coordinating control flow between the model and the user through interrupts.

Method

A virtual context management system with tiered memory (main context vs. external storage), where the LLM issues function/tool calls to move data across tiers and uses interrupts to manage control flow and recursive summarization.

Experimental Setup

Evaluated in two domains where limited context hurts: document analysis (processing documents exceeding the LLM's window) and multi-session chat (agents that remember, reflect, and evolve over long interactions).

Results

The abstract reports qualitative capability - MemGPT analyzes documents far exceeding the underlying LLM's context window and sustains coherent long-term chat - without a single headline benchmark number in the abstract; the verified claim is the OS-paging analogy and LLM-as-memory-manager design.

Limitations

Effectiveness depends on the base model's ability to reliably invoke memory-management functions; the abstract does not quantify overhead or error rates from mis-paging.

Relation to the Field

A foundational LLM-memory work now productized as Letta; contrasts with structure-centric agentic memory like A-MEM and with cross-session sharing protocols like the Mesh Memory Protocol.
