A2A (Agent2Agent) Protocol (Google 2025)

M095Open standard (Google, Apr 2025); donated to the Linux Foundation Jun 2025verified source ↗
A2A is an open peer-to-peer protocol that lets AI agents built on different frameworks and by different vendors discover one another, delegate tasks, and share results without exposing their internal memory, tools, or logic.

Background

Enterprises increasingly run agents built on heterogeneous stacks (LangGraph, CrewAI, Semantic Kernel), and these agents need to collaborate across organizational and framework boundaries.

Problem

There was no common language for one autonomous agent to advertise what it can do and hand off a task to another agent while preserving each side's opacity and security.

Motivation / Analysis

A standard for agent-to-agent collaboration is needed to enable complex cross-vendor workflows without forcing agents to share proprietary internals or converge on one framework.

Key Insight

Treating each agent as an opaque service that publishes a machine-readable capability description and accepts tasks through a defined lifecycle enables secure delegation while keeping internal state private.

Challenges

Supporting long-running and streaming tasks, secure authentication, capability discovery, and framework-agnostic interoperability without leaking internal agent state.

Method

Agents publish Agent Cards (served at /.well-known/agent-card.json) describing skills; peers communicate over HTTP with JSON-RPC 2.0 and Server-Sent Events for streaming, using OAuth2/standard auth, and drive tasks through a defined task lifecycle.

Experimental Setup

Validated by ecosystem adoption rather than benchmarks; launched with 50+ partners and demonstrated cross-framework agent interoperability.

Results

No performance numbers; verified facts: released 9 Apr 2025 with 50+ partners, donated to the Linux Foundation in June 2025 (150+ organizations), Agent Cards at /.well-known/agent-card.json, and explicitly complementary to MCP (agent-to-agent vs agent-to-tool).

Limitations

A2A governs inter-agent messaging and task delegation but does not standardize how an agent accesses its own tools/data (MCP's role), nor does it prescribe internal reasoning or memory.

Relation to the Field

Complements MCP: MCP gives an agent its tools, A2A lets that agent collaborate with peers; sits above MCP in layered interoperability roadmaps and is compared against ACP/ANP in agent-protocol surveys and ProtocolBench.
