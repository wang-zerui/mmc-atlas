RouterBench (Hu et al. 2024)

M028arXiv 2024verified source ↗
The first standardized benchmark for multi-LLM routers, evaluating them on the dollar-cost vs quality frontier using a large corpus of pre-generated inference outcomes.

Background

LLM routing systems proliferated because no single model is best on all tasks under cost constraints, but there was no common yardstick to compare them.

Problem

The absence of a standardized benchmark for evaluating LLM routers hinders progress and makes cross-paper comparison impossible.

Motivation / Analysis

Routers were each evaluated on ad-hoc setups; a shared benchmark with pre-computed outputs enables cheap, reproducible, apples-to-apples router comparison.

Key Insight

Because routers only choose among fixed models, one can pre-generate every model's output for every query once and then evaluate any router offline against that cached matrix.

Challenges

Assembling broad, representative query/model coverage and defining a theoretical framework and metrics that fairly capture the cost-quality trade-off.

Method

RouterBench provides a systematic evaluation framework: a dataset of pre-generated inference outcomes plus a theoretical routing framework and metrics, enabling offline comparative analysis of routing approaches on the cost-quality Pareto frontier.

Experimental Setup

Over 405k pre-generated inference outcomes from representative LLMs; used to compare a range of routing strategies. Specific model list is in the full paper (not in abstract).

Results

Delivers 405k+ pre-generated inference outcomes and a theoretical routing framework plus comparative router analysis (verified in claim_checked); enables reproducible cost-vs-quality router evaluation.

Limitations

Offline pre-generated outputs cannot capture dynamic/agentic or multi-turn routing; fixed model pool and datasets bound its coverage.

Relation to the Field

The reference benchmark for this family and a common evaluation substrate; distinct from and superseded in scale by LLMRouterBench (M122) and complemented by agentic benchmarks like TwinRouterBench (M127).
