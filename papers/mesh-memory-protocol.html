Mesh Memory Protocol (Xu 2026)

M213arXiv 2026 (v0.2.3 specification, CC BY 4.0)verified source ↗
The Mesh Memory Protocol (MMP) is a proposed semantic-infrastructure layer that lets teams of LLM agents share, evaluate, and combine each other's cognitive state across sessions, built from four composable primitives (CAT7, SVAF, inter-agent lineage, remix).

Background

Existing agent protocols cover tool access and task delegation, but there is no standard semantic layer for agents to exchange cognitive state, and multi-agent setups often just run agents in parallel.

Problem

Cross-session agent-to-agent cognitive collaboration is unsolved: agents accept peer content wholesale, cannot trace claims to their source (so echoes of their own thinking look like new evidence), and retain memory based on retrieval rather than principled storage.

Motivation / Analysis

Real collaboration requires agents to selectively integrate peers' reasoning, know where each claim came from, and keep memory that stays relevant across sessions - a missing semantic-infrastructure layer.

Key Insight

Cognitive state should be exchanged as structured blocks that the receiver evaluates field-by-field against its own role-indexed anchors, storing only its own re-evaluated understanding (never the raw peer signal) with full provenance.

Challenges

Designing a fixed schema expressive enough for cognitive state, evaluating peer content per field, tracking provenance across content-hash keys, and preventing self-echo from being mistaken for external evidence.

Method

Four composable primitives: CAT7 (a fixed seven-field schema for each Cognitive Memory Block), SVAF (role-indexed field-by-field acceptance/rejection), inter-agent lineage (provenance via parent/ancestor content-hash keys), and remix (storing only the receiver's role-evaluated understanding).

Experimental Setup

A specification-style paper with reference deployments rather than benchmarks; described as specified, shipped, and running in production across three reference deployments, each session an autonomous mesh peer with its own identity and memory.

Results

No benchmark numbers are reported; the verified contribution is the MMP v0.2.3 spec, its cross-session agent-to-agent cognitive-collaboration framing, the four-primitive design, and reference implementations published on npm (@sym-bot/sym, @sym-bot/mesh-channel).

Limitations

The paper provides no quantitative evaluation of the protocol's effectiveness; claims of production use are described rather than empirically measured.

Relation to the Field

Extends single-agent memory work (MemGPT, A-MEM) to the multi-agent, cross-session setting, and answers the semantic-alignment gap identified by 'Beyond Message Passing' by adding a memory/semantics layer above transport protocols like A2A and MCP.
