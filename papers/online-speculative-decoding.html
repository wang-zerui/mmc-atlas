Online Speculative Decoding (Liu et al. 2023)

M108ICML 2024verified source ↗
Online Speculative Decoding continuously retrains the draft model(s) on the live user-query distribution using knowledge distillation and spare serving-cluster FLOPs, closing the train/serve distribution shift and enabling per-topic specialist drafters.

Background

Speculative decoding accelerates LLM inference by having a small draft model propose tokens that the target model verifies in parallel.

Problem

Its speedup collapses when the draft model is inaccurate, which happens with diverse inputs and a large capability gap between draft and target, because static drafters mismatch the actual query distribution.

Motivation / Analysis

Serving clusters have idle compute; using it to keep the draft model aligned with incoming queries could raise acceptance rates and cut latency without extra hardware.

Key Insight

Continuously updating the draft model on observed user queries via distillation reduces the mismatch between the drafter's training distribution and the real query stream, and even allows multiple topic-specialized drafters.

Challenges

Adapting draft models online without disrupting serving, using spare FLOPs efficiently, and maintaining consistency as the query distribution shifts.

Method

A prototype built on knowledge distillation that continuously updates one or more draft models on observed user query data, adapting to the live distribution during serving.

Experimental Setup

Evaluated on both synthetic and real query data, measuring token acceptance rate and end-to-end latency versus static drafters.

Results

Increases token acceptance rate by 0.1 to 0.65 and reduces latency by 1.22x-3.06x (arXiv abstract reports 1.42x-2.17x) versus static draft models; accepted at ICML 2024.

Limitations

Requires spare serving-cluster compute and an online training pipeline; gains depend on how much the query distribution drifts and the draft/target capability gap.

Relation to the Field

The multi-model efficiency work in this family focused on inference (draft/target collaboration) rather than agent orchestration; complements training-based efficiency methods like Optima and system co-design work like FlexMARL.
