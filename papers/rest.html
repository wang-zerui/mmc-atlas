REST: Retrieval-Based Speculative Decoding (He et al. 2023)

M085NAACL 2024verified source ↗
Draft tokens are retrieved from a datastore of common text patterns instead of generated by a draft model, making acceleration plug-and-play with no training.

Background

Speculative decoding needs a separate draft model that must be trained or carefully matched to the target, adding overhead and limiting portability.

Problem

Provide speculative-style acceleration for any LLM without training or maintaining a draft model.

Motivation / Analysis

Text generation frequently reuses common patterns, so relevant continuations can simply be looked up from stored text given the current context.

Key Insight

Retrieval from a datastore can supply high-quality draft tokens, replacing the draft model entirely and making the method plug-and-play.

Challenges

Building and querying the datastore efficiently; ensuring retrieved drafts have high enough acceptance to yield speedups.

Method

Given the current context, REST retrieves candidate continuations from a datastore (knowledge reservoir), forms draft tokens, and has the target LLM verify them, integrating with any model without additional training.

Experimental Setup

Evaluated on 7B and 13B models for code and text generation in a single-batch setting.

Results

1.62x-2.36x speedup on 7B/13B code and text generation with no training required (verified in claim_checked). The abstract does not explicitly state distribution preservation, but as a draft-verify method it is designed to be lossless; the losslessness claim is not explicit in the abstract.

Limitations

Speedup depends on datastore coverage and domain overlap; less effective when generated text has few reusable patterns; distribution-preservation not explicitly stated in the abstract.

Relation to the Field

A draft-model-free variant of speculative decoding using retrieval rather than a model (like Lookahead's n-grams or self-drafting), complementary to model-based drafters such as EAGLE and Medusa.
