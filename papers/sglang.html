SGLang / RadixAttention (Zheng et al. 2024)

M110arXiv 2023 (NeurIPS 2024)verified source ↗
SGLang pairs a frontend language for structured LLM programs with a runtime whose RadixAttention automatically reuses KV cache across generation calls via an LRU radix tree, reaching up to 6.4x higher throughput.

Background

LLMs are increasingly used for complex tasks needing multiple generation calls, advanced prompting, control flow, and structured inputs/outputs, but systems to program and execute these programs efficiently are lacking.

Problem

Existing inference systems do not exploit the shared prefixes and structure across the many correlated calls in agentic, reasoning, and multi-turn programs, wasting redundant KV-cache computation.

Motivation / Analysis

A co-designed frontend language plus runtime could expose program structure so the runtime can reuse KV cache automatically and decode structured outputs faster.

Key Insight

Storing KV cache in an LRU radix tree (RadixAttention) enables automatic cross-call prefix reuse with cache-aware scheduling, and compressed finite-state machines accelerate structured (e.g. JSON) decoding.

Challenges

Automatically reusing cache without programmer effort, scheduling to maximize cache hits, and handling no overhead on cache misses while supporting parallelism primitives.

Method

A frontend language with generation/parallelism primitives, plus a runtime featuring RadixAttention (radix-tree KV reuse) and compressed FSMs for structured output decoding.

Experimental Setup

Evaluated on agent control, logical reasoning, few-shot benchmarks, JSON decoding, RAG pipelines, and multi-turn chat, across LLMs and multimodal models, versus systems like Guidance and vLLM.

Results

Achieves up to 6.4x higher throughput versus state-of-the-art systems; cache hit rates reported 50-99% with no overhead on miss, though speedups can collapse under poor scheduling / prefix ordering. Verified via abstract and LMSYS blog.

Limitations

Benefits depend heavily on shared prefixes and good scheduling; adversarial or badly ordered request streams reduce cache reuse and erode the speedup.

Relation to the Field

Extends vLLM-style block sharing into automatic radix-tree prefix reuse; directly relevant to multi-call agent workflows (ReAct, ToT, multi-turn) and complements DistServe/Parrot.
