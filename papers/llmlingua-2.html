LLMLingua-2 (Pan et al. 2024)

M090ACL 2024 (Findings)verified source ↗
Distills GPT-4's compression knowledge into a small bidirectional encoder that does prompt compression as token classification, faster and more faithful than LLMLingua.

Background

Task-agnostic prompt compression is valuable but prior methods rely on unidirectional causal-LM information entropy, an indirect metric misaligned with the compression objective.

Problem

Compress prompts in a task-agnostic, faithful, and fast way without depending on entropy from a causal LM.

Motivation / Analysis

Entropy from causal LMs uses only left context and is misaligned with compression; a supervised, bidirectional formulation trained on distilled data should be more faithful and efficient.

Key Insight

Formulating compression as token classification over a full bidirectional context, trained on data distilled from GPT-4, directly optimizes for keeping the right tokens and is far cheaper to run.

Challenges

Constructing an extractive compression dataset by distilling GPT-4; ensuring faithfulness to original content; keeping the compressor small and fast.

Method

GPT-4 is prompted to compress text, forming an extractive compression dataset; a small Transformer encoder (e.g., XLM-RoBERTa-large, mBERT) is trained to classify each token as keep/drop using bidirectional context.

Experimental Setup

Evaluated on in-domain and out-of-domain benchmarks, comparing compression quality, compressor speed, and end-to-end latency against LLMLingua and baselines.

Results

2x-5x compression; the compressor is 3x-6x faster than prior prompt-compression methods and yields 1.6x-2.9x end-to-end latency speedup (verified in claim_checked). The method is lossy (extractive token dropping) but designed to preserve task performance faithfully.

Limitations

Lossy compression; quality bounded by the distilled GPT-4 labels and the small encoder's capacity; extractive-only (cannot rewrite).

Relation to the Field

Directly improves LLMLingua by replacing entropy-based, causal compression with a distilled bidirectional token-classifier; a strong-weak delegation method feeding compressed context to the large model.
