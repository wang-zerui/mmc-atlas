Switchcraft (Agarwal et al. 2026)

M124arXiv 2026verified source ↗
A router built specifically for agentic tool/function calling (not chat): a DistilBERT classifier picks the cheapest correct model inline under a latency budget, and shows bigger models don't reliably win at tool use.

Background

Most routers target open-ended chat quality, but agents increasingly spend their calls on structured tool/function invocation, a different competency.

Problem

Chat-oriented routers don't optimize for tool-calling correctness, and it is unclear whether larger models are even better at tool use.

Motivation / Analysis

Agentic systems need routing that maximizes tool-call correctness while respecting inline latency budgets, a regime prior routers ignore.

Key Insight

For function calling, model size is not a reliable proxy for correctness, so a lightweight classifier can pick the cheapest model that will call the tool correctly.

Challenges

Routing must run inline within an agent's latency budget and generalize across diverse function-calling benchmarks.

Method

Switchcraft trains a DistilBERT classifier that, per query, selects the cheapest model predicted to produce a correct tool call, operating inline under a latency constraint; positioned as the first router for agentic tool calling.

Experimental Setup

Evaluated over 5 function-calling benchmarks; measures tool-calling accuracy and cost against always-use-large baselines.

Results

Achieves 82.9% accuracy at 84% cost reduction and shows larger models do not reliably outperform on tool use (verified in claim_checked); submitted 8 May 2026, preprint (cs.AI/cs.MA).

Limitations

Scope limited to function/tool-calling correctness; DistilBERT classifier bounded by training coverage; no venue yet.

Relation to the Field

Brings routing into the agentic/tool-use setting, complementing agentic step-level benchmark TwinRouterBench (M127) and contrasting with chat-quality routers.
