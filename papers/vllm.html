vLLM / PagedAttention (Kwon et al. 2023)

M109SOSP 2023 (Best Paper)verified source ↗
PagedAttention manages the KV cache like OS virtual-memory paging, giving near-zero fragmentation and ref-counted block sharing within and across requests, which lets vLLM batch more requests and raise serving throughput 2-4x.

Background

High-throughput LLM serving requires batching many requests, but each request's key-value (KV) cache is large and grows and shrinks dynamically during generation.

Problem

Existing serving systems manage KV-cache memory inefficiently, wasting it through fragmentation and duplication, which caps the achievable batch size and thus throughput.

Motivation / Analysis

Classical operating-system virtual memory and paging solve exactly this kind of dynamic, fragmented allocation problem, suggesting the same idea could manage KV cache.

Key Insight

Treating KV cache as non-contiguous fixed-size blocks (pages) mapped through a table enables near-zero memory waste and flexible block sharing, including reference-counted prefix sharing across requests.

Challenges

Adapting attention computation to operate over non-contiguous paged memory, and safely sharing/copying blocks (copy-on-write) across concurrent requests without correctness loss.

Method

PagedAttention stores KV cache in fixed-size blocks with a block table; vLLM builds on it to achieve near-zero KV waste and ref-counted block sharing enabling prefix reuse within and across requests.

Experimental Setup

Evaluated on popular LLMs against state-of-the-art systems FasterTransformer and Orca across sequence lengths, model sizes, and decoding algorithms.

Results

vLLM improves throughput by 2-4x at the same latency versus FasterTransformer and Orca, with larger gains for longer sequences, larger models, and more complex decoding; SOSP 2023 Best Paper. Verified via abstract.

Limitations

PagedAttention adds indirection overhead and requires a custom attention kernel; the reported 2-4x is relative to 2023 baselines and gains vary by workload.

Relation to the Field

Foundational serving substrate that many later systems (SGLang, DistServe, Parrot, PrefillShare) build on or compare against; its block sharing is the precursor to radix/prefix-reuse schemes.
