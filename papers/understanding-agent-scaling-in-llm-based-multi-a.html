Understanding Agent Scaling in LLM-Based Multi-Agent Systems via Diversity (2026)

M157arXiv 2026 (preprint)verified source ↗
An information-theoretic account of why adding agents saturates: homogeneous outputs are correlated so gains cap at task uncertainty, while heterogeneous agents add complementary channels; empirically 2 diverse agents match or exceed 16 homogeneous ones.

Background

Multi-agent LLM systems are scaled by adding agents, but returns often diminish quickly.

Problem

It was unclear why agent scaling saturates and when adding agents is worthwhile.

Motivation / Analysis

A principled, architecture-agnostic account of scaling limits would guide efficient, robust MAS design.

Key Insight

MAS performance is bounded by task uncertainty, not agent count: homogeneous agents produce correlated outputs that plateau early, while diverse agents (varied models/prompts/tools) contribute complementary information channels.

Challenges

Quantifying effective independent contribution without ground-truth labels.

Method

Develops an information-theoretic framework deriving architecture-agnostic improvement bounds via 'effective channel count,' and introduces K*, a label-free metric of the number of effective channels.

Experimental Setup

Empirical scaling studies comparing homogeneous vs heterogeneous agent teams across tasks (code/dataset released).

Results

Reports that 2 diverse agents can match or exceed 16 homogeneous agents, quantified via the K* effective-channel-count metric; homogeneous agents plateau early due to correlated outputs. Confirmed vs abstract; magnitudes not independently audited.

Limitations

arXiv preprint with no venue; specific magnitudes not independently audited and depend on the diversity sources chosen.

Relation to the Field

Provides the information-theoretic explanation behind the Ringelmann-effect ceiling (M156) and reinforces heterogeneity over raw scaling (vs MacNet's agent-count scaling).
