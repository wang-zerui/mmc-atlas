MetaLLM (Nguyen et al. 2024)

M032arXiv 2024pending audit source ↗
MetaLLM frames per-query LLM selection as a multi-armed bandit that optimizes an accuracy-cost reward, choosing the best model without first querying the LLMs.

Background

Available LLMs differ in ability and price, so a fixed default model is suboptimal for classification and multiple-choice QA workloads.

Problem

Select the optimal LLM for each incoming query to balance prediction accuracy against inference cost, under uncertainty about which model will be correct.

Motivation / Analysis

Cascade methods query models sequentially, incurring extra cost; a bandit can learn to pick the right model up front from query features alone.

Key Insight

Routing under an accuracy-cost objective maps naturally onto a multi-armed bandit, where each LLM is an arm and the reward trades correctness against price.

Challenges

Balancing exploration and exploitation across models and defining a reward that meaningfully combines accuracy and cost; magnitudes of savings not audited against primary tables.

Method

MetaLLM formulates model selection as a multi-armed bandit: from query features it selects an LLM (arm) to maximize an accuracy-minus-cost reward, without needing to query the candidate LLMs first.

Experimental Setup

Zero-shot classification and multiple-choice QA using models served via OpenAI, Together AI, and an open-source LLM.

Results

The MAB formulation for zero-shot classification/MCQA is confirmed in the abstract; a 50-70% cost-reduction figure is survey-reported and NOT audited against the paper's primary tables (per claim_checked).

Limitations

Scope limited to classification/MCQA; headline cost-savings numbers are unverified; bandit assumes stationary reward structure.

Relation to the Field

Introduces the bandit view of routing later extended by online-bandit methods (NeuralUCB routing M131, dueling-feedback routing M128); contrasts with supervised routers and cascades.
