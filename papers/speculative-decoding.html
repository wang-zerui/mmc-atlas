Speculative Decoding (Leviathan et al. 2023)

M005ICML 2023 (Oral)verified source ↗
A small drafter proposes tokens that the large target model verifies in parallel, giving exact-distribution speedups without retraining or architecture changes.

Background

Decoding K tokens from a large autoregressive transformer requires K serial forward passes, making inference memory-bandwidth-bound and slow even when hardware parallelism is available.

Problem

Accelerate sampling from a fixed large model without altering its output distribution and without retraining or changing its architecture.

Motivation / Analysis

Hard language-modeling sequences contain many easy subtasks that a cheap approximation model can handle, and speculative execution lets the expensive model run in parallel over the drafter's guesses.

Key Insight

Running the target model in parallel on a drafter's speculative tokens, combined with a novel sampling/acceptance rule, yields multiple tokens per target call while provably matching the target's distribution.

Challenges

Designing an acceptance rule that keeps the output distribution exactly unchanged, and choosing a drafter that is both fast and accurate enough to yield a high acceptance rate.

Method

A smaller approximation model speculatively generates candidate tokens; the target model scores them in parallel; a modified rejection-sampling step accepts a prefix and resamples the first rejected token so the composite sampler equals standard decoding.

Experimental Setup

Evaluated on T5-XXL (11B) against a standard T5X implementation on translation and summarization, using smaller T5 variants as drafters.

Results

2x-3x wall-clock speedup on T5-XXL with identical outputs (verified in claim_checked). The method is distribution-preserving (lossless): it samples from exactly the target model's distribution without changing outputs.

Limitations

Speedup depends on drafter acceptance rate and task difficulty; a poorly matched drafter yields little gain, and extra parallel compute is spent on rejected tokens.

Relation to the Field

The foundational modern draft-verify method; directly extended by Speculative Sampling (Chen et al.), SpecInfer's token trees, EAGLE's feature-level drafting, and self-drafting variants.
