AFlow (Zhang et al. 2024)

M101ICLR 2025 (Oral)verified source ↗
AFlow reframes agentic-workflow design as a search over code-represented workflows and uses Monte Carlo Tree Search to automatically discover high-performing workflows without manual setup.

Background

LLMs solve complex tasks via agentic workflows of detailed instructions and LLM-invoking steps, but these workflows are typically built by hand.

Problem

Constructing agentic workflows demands significant human effort, limiting scalability; prior automation attempts still rely on manual initialization or restricted search spaces.

Motivation / Analysis

Automating the exploration of workflow structures would remove the human bottleneck and let cheaper models rival expensive ones.

Key Insight

Represent workflows as code (LLM-invoking nodes connected by edges) and treat their design as a tree search problem, so MCTS can iteratively refine them using structured experience and execution feedback.

Challenges

The workflow space is vast; the method needs efficient exploration, reusable operators (e.g. Ensemble, Review, Revise), and reliable execution feedback to guide the search.

Method

AFlow performs MCTS over code-represented workflows, iteratively modifying code, accumulating tree-structured experience, and using execution feedback to converge on effective workflows built from reusable operators.

Experimental Setup

Validated across six benchmark datasets, with comparisons to state-of-the-art baselines and to ADAS.

Results

Reports a 5.7% average improvement over state-of-the-art baselines across six benchmarks and enables smaller models to outperform GPT-4o on certain tasks at 4.55% of its inference cost in dollars; the batch note further cites +57% over ADAS on MATH-lv5/MBPP.

Limitations

Gains depend on the quality of the operator library and execution feedback; MCTS over code is compute-intensive and results are reported on six benchmarks that may not cover all task types.

Relation to the Field

A search-based workflow optimizer that outperforms the code-archive search of ADAS and complements graph-based methods like GPTSwarm; part of the automated MAS-design line later extended by OMAC and MAS-Orchestra.
