Medusa (Cai et al. 2024)

M079ICML 2024verified source ↗
Extra lightweight decoding heads on a frozen backbone predict several future tokens in parallel, verified via tree attention; Medusa-1 is lossless while Medusa-2 uses typical-acceptance.

Background

Autoregressive decoding is bottlenecked by repeatedly loading full model weights from HBM per step; speculative decoding helps but requires maintaining a separate draft model.

Problem

Accelerate LLM decoding without the engineering burden of a separate, well-matched draft model.

Motivation / Analysis

Adding a few extra prediction heads to an existing model is far simpler than obtaining and serving a separate drafter, and can still propose multiple future tokens.

Key Insight

Multiple decoding heads on the same backbone can predict several subsequent tokens, and a tree-based attention mechanism lets many candidate continuations be built and verified simultaneously each step.

Challenges

Raising the acceptance rate of head predictions; verifying many candidate branches efficiently; handling cases without training data (solved via self-distillation).

Method

Extra decoding heads predict tokens at positions +1, +2, etc.; tree attention constructs and verifies multiple candidate continuations in one pass. Medusa-1 trains heads on a frozen backbone; Medusa-2 fine-tunes jointly with the backbone plus a typical-acceptance scheme.

Experimental Setup

Evaluated across models of varying sizes and training setups (e.g., Vicuna family) on standard chat/generation benchmarks.

Results

Medusa-1 achieves >2.2x speedup with no quality compromise (lossless, frozen backbone); Medusa-2 reaches 2.3-3.6x via typical-acceptance sampling, which is not strictly distribution-preserving (lossy) though it preserves backbone capabilities (verified in claim_checked).

Limitations

Requires training/fine-tuning the extra heads; Medusa-2's typical acceptance trades strict distribution preservation for higher speedup.

Relation to the Field

A draft-model-free alternative to speculative decoding using self-hosted heads and tree attention; conceptually descends from Blockwise Parallel Decoding and is related to EAGLE's self-drafting.
