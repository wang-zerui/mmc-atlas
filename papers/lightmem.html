# LightMem: Lightweight and Efficient Memory-Augmented Generation ## TL;DR - Compresses incoming experience, consolidates short-term memory by topic, and moves long-term updates off the critical path. ## Problem - Reduce memory-system overhead while preserving useful accumulated experience. ## Insight - Compression can occur at sensory intake, short-term consolidation, and asynchronous long-term update. ## Method - LightMem combines sensory compression, topic-aware short-term memory, and offline long-term consolidation. ## Setup - Memory-agent benchmarks measuring accuracy, token use, API calls, and runtime. ## Results - Reports up to +10.9% accuracy, 117x token, 159x API-call, and 12x runtime improvements in the stated comparisons. ## Challenges - Large efficiency ratios depend on workload and baseline, and delayed updates may make memory stale. ## Limitations - The maxima come from different metrics/settings and should not be read as one simultaneous operating point.

Background

Memory-augmented agents can spend more tokens and calls managing memory than solving the task.

Problem

Reduce memory-system overhead while preserving useful accumulated experience.

Motivation / Analysis

Writing every observation and repeatedly reconsolidating the store is unnecessary.

Key Insight

Compression can occur at sensory intake, short-term consolidation, and asynchronous long-term update.

Challenges

Large efficiency ratios depend on workload and baseline, and delayed updates may make memory stale.

Method

LightMem combines sensory compression, topic-aware short-term memory, and offline long-term consolidation.

Experimental Setup

Memory-agent benchmarks measuring accuracy, token use, API calls, and runtime.

Results

Reports up to +10.9% accuracy, 117x token, 159x API-call, and 12x runtime improvements in the stated comparisons.

Limitations

The maxima come from different metrics/settings and should not be read as one simultaneous operating point.

Relation to the Field

Memory-pipeline compression adjacent to trajectory summarization; it reduces both active state and maintenance work.
