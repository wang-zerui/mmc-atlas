SpecInfer (Miao et al. 2023)

M081ASPLOS 2024verified source ↗
Small speculative models propose a token tree that the LLM verifies in parallel as a tree verifier, provably preserving output quality while cutting latency.

Background

Serving generative LLMs is expensive because incremental (one-token-at-a-time) decoding underuses hardware and dominates latency and compute.

Problem

Reduce LLM serving latency and cost while provably keeping the same generative quality as standard decoding.

Motivation / Analysis

A single linear draft sequence limits how many tokens can be accepted; branching drafts into a tree captures more plausible continuations and raises the accepted-token count per verification.

Key Insight

Organizing multiple small-model guesses into a token tree and having the LLM act as a parallel tree verifier lets many candidate sequences be checked at once, provably preserving model quality.

Challenges

Efficiently verifying an entire token tree in one pass via a novel tree-based parallel decoding mechanism; managing the tree structure across distributed and offloading settings.

Method

Boosted/collectively-tuned small speculative models generate a token tree whose nodes are candidate sequences; a tree-based parallel decoding mechanism has the LLM verify all branches simultaneously and commit the longest accepted path.

Experimental Setup

Evaluated in distributed LLM inference and offloading-based inference settings against existing serving systems, implemented in FlexFlow.

Results

1.5-2.8x faster (distributed) and 2.6-3.5x faster (offloading) than existing serving systems, provably preserving model quality (verified in claim_checked). The method is distribution-preserving (lossless).

Limitations

Requires maintaining and aligning multiple small speculative models; tree verification adds implementation complexity in the serving system.

Relation to the Field

Introduced tree-based speculative verification, later adopted by Medusa and Staged Speculative Decoding; extends the linear draft-verify of Leviathan/Chen to branching token trees.
