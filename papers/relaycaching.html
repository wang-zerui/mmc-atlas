RelayCaching (Geng et al. 2026)

M222arXiv 2026 (cs.LG/cs.AI)pending audit source ↗
RelayCaching is a training-free method that recycles earlier agents' decoding-phase KV caches into later agents' prefill, selectively recomputing only the sparse positions that deviate, to cut cascading context-redundancy overhead in LLM collaboration pipelines.

Background

Growing task complexity has pushed the field from single models toward multi-agent LLM systems, in which later agents re-process content already generated by earlier agents.

Problem

This causes redundant prefill computation for shared, previously-generated content, inflating KV-cache memory and time-to-first-token (TTFT); existing KV-cache reuse methods either lose accuracy on agent-generated outputs or achieve low reuse due to rigid constraints.

Motivation / Analysis

The KV cache for identical content is largely consistent across phases, so decoding-phase caches from one agent could be reused directly in the next agent's prefill.

Key Insight

Prefix-induced deviations between the reused decoding KV cache and a fresh prefill are sparse and localized within a limited range of layers and token positions, so only those positions need recomputation to keep accuracy.

Challenges

Identifying which layers/positions deviate, recomputing only those to bound overhead, and doing so training-free while maintaining accuracy on agent-generated content.

Method

A training-free inference technique that reuses decoding-phase KV caches from previous agents in subsequent agents' prefill, selectively recomputing KV only at the sparse affected positions.

Experimental Setup

Evaluated on collaborative tasks spanning mathematical reasoning, general knowledge, and code generation versus a standard multi-agent pipeline.

Results

Reports over 80% KV-cache reuse and up to 4.7x TTFT reduction versus the standard pipeline with negligible accuracy degradation; arXiv Feb 2026, no venue stated, magnitudes not independently audited.

Limitations

Reuse assumes strong cross-phase KV consistency for shared content; gains depend on how much content is shared across agents; no peer-reviewed venue yet and numbers unaudited.

Relation to the Field

Training-free counterpart to PrefillShare's trained cross-model prefill sharing; extends prefix KV reuse (vLLM/SGLang) to the case where the reused content was generated by a different agent.
