技术路线图

本研究的核心结论是:在算力对齐(compute-matched)评测下,多数多模型协作的收益会消失——等 token 预算下单个强模型可匹配或超过多智能体系统(M115),组合被 co-failure 下限 β 封顶(M135),辩论很少胜过 self-consistency(M059)。稳健的收益只来自结构性保证(无损耦合)或有效控制信号(校准路由、诚实的异质性、可靠验证)。本 ROADMAP 把这个不对称转化为工程计划:不再为不划算的协作买单,只在保证/信号真实的地方投入。

H1
当下 → 近期

诚实度量,收割有保证的确定收益

把 compute-matched 评测设为默认,只部署可证明或近乎普适净正收益的机制。

  • 把'对单个最强模型的 FLOPs/token 对齐评测'定为汇报标准(不再要 token 混淆出来的收益)。[M115, M135, M217]

  • 全面上线 serving 层 KV/prefix 复用(vLLM/SGLang/Parrot/DistServe 及 2026 多智能体复用)——近乎普适的确定收益。注意会破坏 judge 的例外。[M109-M112, M221-M223, M226]

  • 在延迟敏感处使用无损强弱耦合(speculative decoding 家族)——唯一保分布的协作;有损变体标为明确取舍。[M190, M192;注意 M191/M193/M194]

  • 仅在存在有效信号时开启校准的、预算/工具感知的路由;否则默认单个最强模型。[M121, M123, M124, M125;受 M122/M135 约束]

成功判据 每条协作结论都在匹配算力下汇报;serving 复用 + 路由带来可测成本下降且不掉质量。
H2
中期

让控制信号可靠、让边界透明

攻克协作不划算的根因:不可靠的门控、不知何时该组合、以及 harness 绑定。

  • 构建 co-failure 下限 β 的廉价估计器,使'何时该组合'可事前判定而非事后。[M135]

  • 加固验证:异质family judge panel + 训练/agentic verifier,并在困难对抗样本上评测(agreement κ 会高估质量)。[M070/M072/M075/M078, M181, M077, M185-M187]

  • 做一个与 harness 无关的 model gateway(经 base_url 在 model-call 边界介入),让收益跨 Claude Code/Codex/OpenHands 迁移而无需逐个重建 harness。[C 线]

  • 把 turn 次数当一等成本轴:在网关处把低价值、缓存读沉重的 turn 路由给免费本地模型;度量真实(非账面上界)节省与 false-accept。[C 线, M013, M226]

  • 训练 RL 的'何时停止'编排决策,终止过长的 agent loop。[M165]

成功判据 β 可在预算内估计;困难任务上 verifier false-accept ≈ 0;网关在 2+ harness 上于匹配成功率下降低强模型成本。
H3
远期

一套'协作何时取胜'的预测性理论

从个案收益走向对互补性、自修复与语义协调的预测性刻画。

  • 异质性的预测理论:给定任务 + 模型池,在运行前预测互补 vs 负协同。[M065, M114, M118, M156, M160]

  • 补上 agent 协议的语义对齐空白:把澄清/修复做成一等公民,而非 prompt 层的技术债。[M097, M212, M320]

  • 可扩展的失败归因:自动定位是哪个 agent/哪一步导致结果,闭合 协作→归因→演化 的环。[M023, M227, M228, M314]

  • 由归因驱动重组结构的自演化系统——以 compute-matched 门槛约束,使演化配得上其成本。[M314, M321]

成功判据 互补性可预测;协议承载语义;失败可自动归因;自演化在匹配算力下呈净收益。

贯穿主线

贯穿三个时间层的不变量始终一致:只在结构性保证或有效信号使其在匹配算力下净正收益的地方,才为协作买单——并且一切都以单个最强模型为基线来度量。
