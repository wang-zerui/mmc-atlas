# Scaling Long-Horizon LLM Agent via Context-Folding ## 一句话 - 把子任务分支到独立上下文中执行，完成后只把紧凑的返回摘要折叠回父轨迹。 ## 问题 - 避免已完成的子任务轨迹永久占据父上下文。 ## 洞见 - 把子轨迹视为有作用域的调用帧，只向父轨迹返回结果。 ## 方法 - Context-Folding 在独立上下文执行分支，返回紧凑摘要，并用 FoldGRPO 训练折叠行为。 ## 设置 - 在长程 agent 基准上比较平铺轨迹、手工压缩与学习式折叠。 ## 结果 - 论文报告活跃上下文约缩小 10 倍，并改善长程扩展能力。 ## 挑战 - 选择折叠边界，并生成足以支持后续验证的返回状态。 ## 局限 - 返回摘要有损；若没有证据或验证信号，隐藏分支的错误可能被保留下来。

背景

长程 agent 把局部子任务细节与全局任务状态混在同一序列中。

问题

避免已完成的子任务轨迹永久占据父上下文。

动机 / 分析

许多中间工具调用只在局部分支运行期间有价值。

关键洞见

把子轨迹视为有作用域的调用帧，只向父轨迹返回结果。

挑战

选择折叠边界，并生成足以支持后续验证的返回状态。

方法

Context-Folding 在独立上下文执行分支，返回紧凑摘要，并用 FoldGRPO 训练折叠行为。

实验设置

在长程 agent 基准上比较平铺轨迹、手工压缩与学习式折叠。

结果

论文报告活跃上下文约缩小 10 倍，并改善长程扩展能力。

局限

返回摘要有损；若没有证据或验证信号，隐藏分支的错误可能被保留下来。

与领域的关系

轨迹压缩中最清晰的宏观轮次 branch-and-return 形式。
