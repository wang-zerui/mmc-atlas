# MemGPT ## 一句话 - MemGPT 将操作系统的内存层级思想应用于 LLM，让模型通过 function call 自我管理分层 memory——固定的 main context（类似 RAM）加上外部存储（类似 disk）——从而表现得仿佛拥有远大于实际的 context window。 ## 问题 - 这种有界的 context 使 LLM 无法维持长对话或对超出 window 的文档进行推理，限制了它在长时程任务中的实用性。 ## 洞见 - LLM 自身可以充当自己的 memory 管理器：赋予它在 main context 与外部存储之间分页数据的工具以及管理控制流的中断，它就能决定保留、驱逐和检索什么，从而营造出无界 context 的假象。 ## 方法 - 一个虚拟 context 管理系统，采用分层 memory（main context 对比外部存储），LLM 通过发出 function/tool call 在各层级间移动数据，并使用中断来管理控制流和递归摘要。 ## 设置 - 在两个受限 context 影响明显的领域中评估：文档分析（处理超出 LLM window 的文档）和多会话聊天（能在长期交互中记忆、反思并演进的 agent）。 ## 结果 - 摘要报告的是定性能力——MemGPT 能分析远超底层 LLM context window 的文档，并维持连贯的长期聊天——摘要中没有单一的标志性 benchmark 数字；已验证的主张是 OS 分页类比和“LLM 作为 memory 管理器”的设计。 ## 挑战 - 教会模型何时将信息分页进/出、在不丢失关键事实的前提下递归摘要，以及通过中断在模型与用户之间协调控制流。 ## 局限 - 其有效性取决于基础模型可靠调用 memory 管理函数的能力；摘要没有量化误分页带来的开销或错误率。

背景

LLM 拥有固定的 context window，限制了它一次能关注的对话历史或文档文本的数量。

问题

这种有界的 context 使 LLM 无法维持长对话或对超出 window 的文档进行推理，限制了它在长时程任务中的实用性。

动机 / 分析

传统操作系统早已通过虚拟内存和在各层级间分页解决了快速内存有限的问题，这暗示了针对 LLM context 的类似解决方案。

关键洞见

LLM 自身可以充当自己的 memory 管理器：赋予它在 main context 与外部存储之间分页数据的工具以及管理控制流的中断，它就能决定保留、驱逐和检索什么，从而营造出无界 context 的假象。

挑战

教会模型何时将信息分页进/出、在不丢失关键事实的前提下递归摘要，以及通过中断在模型与用户之间协调控制流。

方法

一个虚拟 context 管理系统，采用分层 memory（main context 对比外部存储），LLM 通过发出 function/tool call 在各层级间移动数据，并使用中断来管理控制流和递归摘要。

实验设置

在两个受限 context 影响明显的领域中评估：文档分析（处理超出 LLM window 的文档）和多会话聊天（能在长期交互中记忆、反思并演进的 agent）。

结果

摘要报告的是定性能力——MemGPT 能分析远超底层 LLM context window 的文档，并维持连贯的长期聊天——摘要中没有单一的标志性 benchmark 数字；已验证的主张是 OS 分页类比和“LLM 作为 memory 管理器”的设计。

局限

其有效性取决于基础模型可靠调用 memory 管理函数的能力；摘要没有量化误分页带来的开销或错误率。

与领域的关系

一项基础性的 LLM-memory 工作，如今被产品化为 Letta；与以结构为中心的 agentic memory（如 A-MEM）以及跨会话共享 protocol（如 Mesh Memory Protocol）形成对照。
