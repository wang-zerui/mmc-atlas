Cross-Family Speculative Decoding via UAG (Fonal 2026)

M194arXiv 2026已核实 原文 ↗
一项关于 cross-family、cross-tokenizer speculative decoding 的经验性研究（基于 Mistral 的 Bielik-11B 作为 target，搭配 Qwen/Llama drafter），在 Apple Silicon 上通过 Universal Assisted Generation 的 token 翻译实现。

背景

在 tokenizer 不匹配、消费级 unified-memory 硬件上进行 cross-family speculative decoding，相比高带宽 GPU 上的同 tokenizer 场景尚未被充分探索。

问题

判定 cross-family/cross-tokenizer speculative decoding 是否能为在 Apple Silicon 上运行的波兰语 LLM 带来真实加速。

动机 / 分析

实践者可能希望将强 target 与任意便捷的小 drafter 配对，而不论其族系/tokenizer，尤其在消费级硬件上。

关键洞见

tokenizer 间 context-aware 的 token 翻译能提升接受率，但在受 memory-bandwidth 限制的 unified memory 上，理论上的验证节省无法摊销，因为两个模型都受 memory 限制。

挑战

在不匹配词表间翻译 token（naive 与 context-aware）；克服 memory-bandwidth 限制——此时串行 drafting 相对批量验证代价高昂。

方法

用 Universal Assisted Generation (UAG) 扩展 MLX-LM 框架以在 Apple Silicon 上做 cross-tokenizer speculative decoding；target 为 Bielik-11B-Instruct（基于 Mistral），drafter 为 Bielik-1.5B（基于 Qwen）、Qwen2.5-1.5B 与 Llama-3.2-1B，跨三个波兰语数据集与 draft 长度 k in {2,4,6}，对比 naive 与 context-aware 的 token 翻译。

实验设置

三个波兰语数据集（Wikipedia、pl_alpaca、synthetic），跨 draft 长度与 drafter，测量接受率与吞吐。

结果

在结构化文本上至多 1.7x 加速，但在多样化指令上失败；context-aware 翻译持续提升接受率；出人意料的是，专门面向波兰语的 1.5B drafter 表现不及通用的 Qwen2.5/Llama drafter；验证成本在受 memory 限制的 unified memory 上无法摊销（已在 claim_checked 中核实）。在 token 翻译下 losslessness 未被确立（很可能 lossy）。

局限

收益依赖内容、在多样化指令上失败；unified-memory 带宽限制了验证摊销；cross-tokenizer 翻译下 losslessness 未获确认。

与领域的关系

Timor et al. (TLI) 的 lossless heterogeneous-vocabulary SD 算法在消费级硬件上的一个经验性对应研究；二者都面向 cross-tokenizer drafting，但本研究不保证 losslessness。
