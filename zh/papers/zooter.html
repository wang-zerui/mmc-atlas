# Zooter ## 一句话 - Zooter 将奖励蒸馏到训练查询上，训练出一个轻量 router，把每个查询发给最擅长的 LLM，从而避免在推理时运行奖励模型的开销。 ## 问题 - 现有的集成方法在推理时用奖励模型对候选答案排序，这在计算上很昂贵，因为必须查询并评分每一个模型。 ## 洞见 - 奖励模型在训练查询上给出的奖励，可以蒸馏进一个仅依赖查询的路由函数，因此推理时只需一个便宜的 router 即可预测最佳模型，无需任何奖励排序。 ## 方法 - 奖励引导的路由：奖励模型为训练查询的答案打分，其奖励通过 KL 蒸馏进一个 BERT 式的路由函数，并用基于 tag 的标签增强对监督去噪；推理时只运行 router，仅增加极小开销。 ## 设置 - 一个涵盖多领域/任务、由 26 个子集组成的综合基准集合；与最佳单模型以及奖励模型排序的集成方法比较。 ## 结果 - 平均上优于最佳单模型，并在 26 个基准子集中的 44% 上排名第一，采用了 KL 奖励蒸馏加基于 tag 的标签增强（已在 claim_checked 中核实），同时以远低的开销超越了多种奖励排序方法。 ## 挑战 - 用作银标监督的奖励信号带有噪声；论文引入基于标签（tag）的标签增强来缓解蒸馏标签中的不确定性。 ## 局限 - 路由质量受限于用于蒸馏的奖励模型；尽管有增强，银标签仍带有噪声。

背景

现成的 LLM 在不同领域各有互补的强项，因此在它们之间做集成/路由能够超越任何单一模型。

问题

现有的集成方法在推理时用奖励模型对候选答案排序，这在计算上很昂贵，因为必须查询并评分每一个模型。

动机 / 分析

如果能仅凭查询就预测出擅长的模型，路由便可避免生成并评分所有候选答案，从而大幅削减开销。

关键洞见

奖励模型在训练查询上给出的奖励，可以蒸馏进一个仅依赖查询的路由函数，因此推理时只需一个便宜的 router 即可预测最佳模型，无需任何奖励排序。

挑战

用作银标监督的奖励信号带有噪声；论文引入基于标签（tag）的标签增强来缓解蒸馏标签中的不确定性。

方法

奖励引导的路由：奖励模型为训练查询的答案打分，其奖励通过 KL 蒸馏进一个 BERT 式的路由函数，并用基于 tag 的标签增强对监督去噪；推理时只运行 router，仅增加极小开销。

实验设置

一个涵盖多领域/任务、由 26 个子集组成的综合基准集合；与最佳单模型以及奖励模型排序的集成方法比较。

结果

平均上优于最佳单模型，并在 26 个基准子集中的 44% 上排名第一，采用了 KL 奖励蒸馏加基于 tag 的标签增强（已在 claim_checked 中核实），同时以远低的开销超越了多种奖励排序方法。

局限

路由质量受限于用于蒸馏的奖励模型；尽管有增强，银标签仍带有噪声。

与领域的关系

一个开创性的奖励蒸馏 router；在概念上与 RouterDC（对比式路由）及后续的基于查询的 router 相关，并与顺序查询模型的 cascade 方法形成对照。
