Anthropic Multi-Agent Research System (2025)

M013Anthropic engineering blog (2025)已核实 原文 ↗
一个 orchestrator-worker 式的研究系统：一个 Claude Opus 4 主导 agent 派生出多个并行的 Claude Sonnet 4 子 agent，各自独立检索并返回经过筛选的发现以供综合。

背景

开放式研究查询需要探索多个方向，无法像静态 RAG 检索那样预先规划。

问题

拥有单一上下文窗口的单个 agent 是广域研究的瓶颈：它无法并行探索众多方面，也难以有效地扩展 token 预算。

动机 / 分析

将子任务委派给拥有独立上下文窗口的并行子 agent，可让系统动态地发现并适应相关信息。

关键洞见

研究性能的大部分方差由总 token 预算驱动；在多个子 agent 间并行化探索，既提升质量，又降低了复杂查询的时延。

挑战

协调子 agent、避免重复劳动、进行来源溯源，以及控制运行大量 agent 所带来的高昂 token 成本。

方法

主导 agent 分析查询、制定策略，并派生子 agent 并行运行各自的检索；主导 agent 综合返回的发现，并判断是否需要进一步研究，最后由一个 CitationAgent 将各项论断归因到来源。

实验设置

内部研究评测以及 BrowseComp benchmark；针对 token 用量、工具调用次数与模型选择的消融实验。

结果

Opus-4 主导 + Sonnet-4 子 agent 的系统在内部研究评测上比单 agent 的 Opus 4 高出 90.2%；在 BrowseComp 上，三个因素解释了 95% 的方差，其中仅 token 用量就约占 80%；多 agent 消耗的 token 约为聊天场景的 15x；并行化最多可将研究时间缩短 90%。

局限

鉴于约 15x 的 token 成本，仅在高价值任务上才具经济可行性；不适用于子 agent 需要共享状态的紧耦合任务。

与领域的关系

orchestrator-worker 角色专业化在生产规模上的一个实例，与 Magentic-One 的 manager-worker 设计高度并行，并推动了在编排轨迹上做 RL 的相关工作。
