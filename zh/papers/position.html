# Position: Collaborative Agentic AI Needs Interoperability Across Ecosystems ## 一句话 - 这篇 ICML 2026 立场论文主张，协作式 agentic AI 需要最小化的共享标准以避免碎片化为互不兼容的生态系统，并提出以四个组件构建的“Web of Agents”作为最小化的架构基础。 ## 问题 - 独立、缺乏协调的开发正将该领域推向一个由碎片化、互不兼容的 agent 生态系统构成的格局。 ## 洞见 - 互操作性应复用现有的标准和基础设施，而非重新发明它们，并且可以由一小组架构组件来刻画，而不需要一个重量级的新技术栈。 ## 方法 - 提出“Web of Agents”，一个最小化的架构基础，包含四个组件：agent 到 agent 的消息传递、交互互操作性、状态管理和 agent 发现，并刻意在可能之处采用现有标准。 ## 设置 - 一篇立场论文；它呈现的是论点和架构，而非实证实验或 benchmark。 ## 结果 - 没有定量结果；已验证的贡献是互操作性论点和四组件的 Web of Agents 架构，将其定位为迈向可互操作 agentic 系统的第一个关键步骤。已被 ICML 2026 接收。 ## 挑战 - 定义一个既足够最小以便被广泛采用、又足够完整以支持真正的跨生态系统协作的基础，涵盖消息传递、交互、状态和发现。 ## 局限 - 作为一篇立场论文，它论证并勾勒了一个架构，但没有实现或对其进行实证验证。

背景

协作式 agentic AI——即 agent 在数字环境中自主感知、规划和行动的系统——预计将重塑各行各业，但各类解决方案正在被独立开发。

问题

独立、缺乏协调的开发正将该领域推向一个由碎片化、互不兼容的 agent 生态系统构成的格局。

动机 / 分析

现在就采用最小化的共享标准，是在碎片化根深蒂固之前构建开放、安全、web 规模且被广泛采用的 agentic 生态系统的必要条件。

关键洞见

互操作性应复用现有的标准和基础设施，而非重新发明它们，并且可以由一小组架构组件来刻画，而不需要一个重量级的新技术栈。

挑战

定义一个既足够最小以便被广泛采用、又足够完整以支持真正的跨生态系统协作的基础，涵盖消息传递、交互、状态和发现。

方法

提出“Web of Agents”，一个最小化的架构基础，包含四个组件：agent 到 agent 的消息传递、交互互操作性、状态管理和 agent 发现，并刻意在可能之处采用现有标准。

实验设置

一篇立场论文；它呈现的是论点和架构，而非实证实验或 benchmark。

结果

没有定量结果；已验证的贡献是互操作性论点和四组件的 Web of Agents 架构，将其定位为迈向可互操作 agentic 系统的第一个关键步骤。已被 ICML 2026 接收。

局限

作为一篇立场论文，它论证并勾勒了一个架构，但没有实现或对其进行实证验证。

与领域的关系

与 Ehtesham 等人 protocol 综述共享反碎片化的动机，并通过主张在 A2A 和 MCP 等具体 protocol 之上设立一个最小化的共享标准层来对其形成补充。
