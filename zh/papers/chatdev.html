# ChatDev ## 一句话 - ChatDev 是一个由聊天驱动的软件开发框架，角色专业化的 LLM agent（CEO/CTO/程序员/测试员）沿着一条“chat chain”协作，并借助 communicative dehallucination 端到端地构建软件。 ## 问题 - 阶段专用的设计会在设计、编码与测试之间造成技术上的不一致，使开发过程碎片化。 ## 洞见 - “chat chain”规范了 agent 沟通的内容（将每个阶段划分为有序的子任务对话），而“communicative dehallucination”规范了它们沟通的方式以捕捉并纠正错误；自然语言有助于设计，编程语言的沟通则有助于调试。 ## 方法 - 由 LLM 驱动、担任专业化角色的 agent（CEO、CTO、程序员、评审员、测试员）以组织成 chat chain 的多轮角色扮演对话，依次推进设计、编码与测试，并配有一套去幻觉（dehallucination）协议以实现可靠交流。 ## 设置 - 在自主软件生成任务上进行评估；对完整应用生成的案例研究，并报告成本与耗时。 ## 结果 - 已核实的贡献：chat-chain + communicative-dehallucination 设计；ChatDev 能在 7 分钟以内、花费不到 $1 完成一个完整的软件应用（标志性效率数字来自论文/ACL 2024）。 ## 挑战 - 在保持流水线自主运行的同时，防止编码幻觉与不一致跨阶段传播。 ## 局限 - 所生成的软件规模较小，且可能存在 bug；角色脚本与瀑布式结构是固定的，而非自适应组合的。

背景

软件开发需要具备不同技能的人协作，但此前的深度学习方法针对的是瀑布模型中各个独立阶段，且各自采用不同的设计。

问题

阶段专用的设计会在设计、编码与测试之间造成技术上的不一致，使开发过程碎片化。

动机 / 分析

以自然语言作为贯穿所有阶段的统一媒介，可保持流程的连贯性，并让单一框架覆盖整个生命周期。

关键洞见

“chat chain”规范了 agent 沟通的内容（将每个阶段划分为有序的子任务对话），而“communicative dehallucination”规范了它们沟通的方式以捕捉并纠正错误；自然语言有助于设计，编程语言的沟通则有助于调试。

挑战

在保持流水线自主运行的同时，防止编码幻觉与不一致跨阶段传播。

方法

由 LLM 驱动、担任专业化角色的 agent（CEO、CTO、程序员、评审员、测试员）以组织成 chat chain 的多轮角色扮演对话，依次推进设计、编码与测试，并配有一套去幻觉（dehallucination）协议以实现可靠交流。

实验设置

在自主软件生成任务上进行评估；对完整应用生成的案例研究，并报告成本与耗时。

结果

已核实的贡献：chat-chain + communicative-dehallucination 设计；ChatDev 能在 7 分钟以内、花费不到 $1 完成一个完整的软件应用（标志性效率数字来自论文/ACL 2024）。

局限

所生成的软件规模较小，且可能存在 bug；角色脚本与瀑布式结构是固定的，而非自适应组合的。
