DeePEn (Huang et al. 2024)

M037NeurIPS 2024已核实 原文 ↗
DeePEn 是一种免训练的集成方法，通过将异质 LLM 逐步的概率分布映射到一个共享的「relative representation」空间来解决词表不匹配问题，从而实现融合。
思维导图

背景

LLM 各有互补优势，促使人们进行集成，但以往方法依赖于额外的奖励/融合模型，且仅作用于文本输出，对未见数据的泛化能力较差。

问题

异质 LLM 使用不同的词表，因此它们逐 token 的概率分布无法直接平均，而纯文本的集成又忽略了内部表示的信息。

动机 / 分析

在概率分布层面进行融合能捕捉到比对最终文本投票更丰富的信息，而且只要能解决词表不匹配问题，就无需训练任何融合模块。

关键洞见

relative representation 理论可将每个模型的分布映射到一个通用的相对空间，在该空间中即使词表不同，聚合也是有意义的。

挑战

对齐异质的 token 词表，将聚合后的分布变换回可用的 next-token 分布，并将延迟开销保持在低水平。

方法

在每个解码步骤，将每个 LLM 的概率分布映射到一个通用的相对空间中进行聚合，然后通过基于搜索的逆变换将其映射回所选主模型的概率空间以选取下一个 token；全程免训练。

实验设置

在三种集成配置（不同 LLM 数量、不同架构、LLM 与专家模型配对）下，跨涵盖学科考试、推理和知识的六个基准进行评测。

结果

在六个基准上取得一致的增益；一个强专家模型甚至能从较弱的 LLM 中获益，DeePEn 与投票互补，且带来的句子级延迟开销低于 2%（已在摘要中确认）。

局限

需要访问逐步的概率分布（不适用于仅有黑盒 API 的模型），且相对空间映射与逆向搜索相比简单投票增加了额外机制。

与领域的关系

一种与 PackLLM (M038) 和 FuseLLM (M039) 并列的分布级融合方法；与 MoA (M001) 和 LLM-Blender (M009) 的文本/答案级聚合形成对比；在集成综述 (M041) 中被归入「during-inference」分支。
