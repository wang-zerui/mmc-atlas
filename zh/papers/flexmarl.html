FlexMARL: Rollout-Training Co-Design for Multi-Agent RL (2026)

M215arXiv 2602.09578 (Feb 2026)待审计 原文 ↗
FlexMARL 是一个端到端框架，对 rollout、训练及其编排进行整体协同设计，用于大规模基于 LLM 的多智能体强化学习，解决同步屏障、rollout 负载不均以及训练资源利用不足等问题。

背景

面向 LLM 的 MARL 在算法层面已有进展，但大规模 MARL 的网络化训练基础设施仍未被充分探索。

问题

现有 RL 训练框架为单智能体情形设计，无法处理 MARL 特有的系统挑战：同步屏障、倾斜的 rollout 负载以及闲置的训练资源。

动机 / 分析

一个联合优化 rollout 与训练的解耦系统，可以消除这些瓶颈，并将基于 LLM 的 MARL 扩展到生产集群。

关键洞见

将 rollout、训练与编排视为一个协同设计问题——借助经验存储与异步微批流水线——可在保持一致性的同时消除同步屏障，而以 agent 为中心的资源分配则消除了负载不均。

挑战

在不破坏一致性的前提下消除同步屏障、平衡高度倾斜的 rollout 请求模式，并在训练中在互相交换状态的 agent 之间分配硬件。

方法

一种解耦架构，包含一个用于数据流的联合编排器、一个经验存储加微批异步流水线以消除同步屏障、一个具备并行采样与分层负载均衡的 rollout 引擎，以及一个具备以 agent 为中心的按需硬件绑定与位置无关通信的训练引擎。

实验设置

在大规模生产集群上评估，与现有训练框架对比。

结果

报告相较现有框架最高 7.3x 的加速和最高 5.6x 的硬件利用率提升；批注指出这些量级未经独立审计。

局限

未注明经同行评审的发表场所，且所报告的加速/利用率数字未经独立审计；增益在特定生产集群配置上测得。

与领域的关系

作为 MAPoRL 与 Maestro 等算法式 MARL 协同训练工作的系统/基础设施对应物，聚焦训练-服务栈而非学习目标。
