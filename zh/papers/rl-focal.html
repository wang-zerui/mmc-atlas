RL-Focal: Dynamic Optimization of LLM Ensembles (Tekin et al. 2026)

M218ICML 2026 (Poster)已核实 原文 ↗
RL-Focal 是一个两阶段强化学习框架，其中 Decider agent 为每个查询挑选一个小而误差多样的集成，Fusion agent 学习化解跨 LLM 的推理冲突，并由一个 focal-diversity 指标引导。

背景

跨多个 LLM 的集成与路由可以超越任何单一模型，但为每个查询选择组合哪些模型是一个困难的组合优化问题。

问题

静态集成浪费算力，且可能把正确答案"平均"掉；为每个查询选出一个小而恰当的集成并有效融合它们并非易事。

动机 / 分析

动态选择误差多样的模型并学习调和其冲突的推理，应能同时胜过单一模型与固定集成。

关键洞见

将问题分解为：一个 Decider 从 N 个模型中选出大小为 m 的小集成以优化误差多样性与性能，以及一个 Fusion agent 学习化解所选模型间的推理冲突；一个建模误差相关性的 focal-diversity 指标可修剪集成组合并改善泛化。

挑战

从组合式候选池中选出优良集成、让融合适配变化的集成组合，并建模误差相关性以避免冗余的模型选择。

方法

两阶段 RL：(1) 一个 Decider RL-agent 通过任务自适应奖励为每个查询动态选择小集成，使用基于误差相关性的 focal-diversity 指标；(2) 一个 Fusion RL-agent 组合所选模型并学习化解其推理冲突。

实验设置

在五个 benchmark 上评估，与池中最佳单一 LLM 及其他集成/路由基线对比。

结果

在五个 benchmark 上使用小规模集成相较最佳单一 LLM 取得 8.48% 的性能提升，并报告了更强的鲁棒性。

局限

增益取决于模型池的多样性与质量；两阶段 RL 增加了训练开销，结果在五个 benchmark 上报告。

与领域的关系

一种动态 LLM 集成/路由优化器，与拓扑优化器（GPTSwarm、HieraMAS）以及推理效率路由互补；是逐查询模型路由在集成选择上的对应物。
