# FrugalGPT ## 一句话 - 一种可学习的 LLM cascade，将每个查询从最便宜的模型开始逐级向上路由，用质量评分器判断当前答案是否已足够好，从而以极低的成本达到 GPT-4 的水平。 ## 问题 - 在拥有异构付费 API 模型池的前提下，如何在保持（甚至提升）答案质量的同时降低 LLM 推理的金钱成本。 ## 洞见 - 一个事后的质量/可靠性估计器可以判断便宜模型的答案是否可接受，从而构建按成本升序、仅在需要时才升级的 cascade。 ## 方法 - FrugalGPT 是 LLM cascade 的一个具体实例：它针对每个任务学习一个 LLM 序列以及一个对每条生成答案打分的评分函数；查询依次流经越来越昂贵的模型，并在第一个分数超过学习阈值的答案处停止。它与提示自适应、LLM 近似共同构成综述中的三种降本策略。 ## 设置 - 在包括 GPT-4、ChatGPT 和 J1-Jumbo 在内的商用 API 上，于 HEADLINES 等 QA/分类数据集上评估；基线是对所有查询都使用最佳单模型（GPT-4）。 ## 结果 - 在成本最多降低 98% 的情况下达到 GPT-4 的性能，或在等成本下将准确率相对 GPT-4 提升 4%。在 HEADLINES 上仅有 16.6% 的查询被路由到 GPT-4（已在 claim_checked 中核实）。 ## 挑战 - 学习该使用哪些模型组合并设定可靠的接受阈值，使提前停止不会牺牲准确率；同时要以低成本、高精度地估计答案质量。 ## 局限 - 质量评分器与 cascade 顺序是按任务/数据集学习的，可能难以迁移；节省幅度取决于查询难度分布以及可用模型之间的价格差距。

背景

商用 LLM API（GPT-4、ChatGPT、J1-Jumbo 等）之间的价格差异高达两个数量级，在大规模场景下对每个查询都直接调用最强模型的开销极其庞大。

问题

在拥有异构付费 API 模型池的前提下，如何在保持（甚至提升）答案质量的同时降低 LLM 推理的金钱成本。

动机 / 分析

以往工作主要优化单模型的提示或近似手段，却没有系统性地利用一个事实：很多查询用便宜模型就能正确回答，昂贵模型只需留给困难样本。

关键洞见

一个事后的质量/可靠性估计器可以判断便宜模型的答案是否可接受，从而构建按成本升序、仅在需要时才升级的 cascade。

挑战

学习该使用哪些模型组合并设定可靠的接受阈值，使提前停止不会牺牲准确率；同时要以低成本、高精度地估计答案质量。

方法

FrugalGPT 是 LLM cascade 的一个具体实例：它针对每个任务学习一个 LLM 序列以及一个对每条生成答案打分的评分函数；查询依次流经越来越昂贵的模型，并在第一个分数超过学习阈值的答案处停止。它与提示自适应、LLM 近似共同构成综述中的三种降本策略。

实验设置

在包括 GPT-4、ChatGPT 和 J1-Jumbo 在内的商用 API 上，于 HEADLINES 等 QA/分类数据集上评估；基线是对所有查询都使用最佳单模型（GPT-4）。

结果

在成本最多降低 98% 的情况下达到 GPT-4 的性能，或在等成本下将准确率相对 GPT-4 提升 4%。在 HEADLINES 上仅有 16.6% 的查询被路由到 GPT-4（已在 claim_checked 中核实）。

局限

质量评分器与 cascade 顺序是按任务/数据集学习的，可能难以迁移；节省幅度取决于查询难度分布以及可用模型之间的价格差距。

与领域的关系

本家族中的奠基性 cascade 方法；后续的路由/cascade 工作（AutoMix、Hybrid LLM、RouterBench、Gatekeeper、UCCI）都在其成本升序、置信度门控的延迟思想之上构建或以其为基准。
