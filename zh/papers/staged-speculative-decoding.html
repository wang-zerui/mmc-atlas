Staged Speculative Decoding (Spector & Re 2023)

M082ICML 2023 ES-FOMO Workshop已核实 原文 ↗
将 speculative batch 重构为 tree，并增加第二个 speculation 阶段（对 draft 本身进行 speculation），以 lossless 方式加速小批量、端侧推理。

背景

小批量与端侧 LLM 推理的算术强度低，因此解码受 memory 限制，难以充分利用硬件并行。

问题

在完美保持输出质量的前提下，为小批量/端侧场景最大化单批解码加速。

动机 / 分析

线性 speculative batch 浪费了并行预算；既重构 batch，又增加一层 speculation，都能提升每批预期接受的 token 数。

关键洞见

tree-structured speculation 提升每批的预期 token 数，而对 draft 模型本身进行 speculative 解码（第二阶段）进一步降低总生成成本。

挑战

在不破坏 exactness 的前提下组合两个 speculation 阶段；在有限的 batch 预算下设计 tree 结构以最大化接受 token。

方法

先将 speculative batch 重构为 tree 以提升预期接受 token 数；再对 draft 模型施加第二阶段的 speculative decoding，加速 drafting 步骤本身，全程针对 target 进行验证。

实验设置

在一个 762M 参数的 GPT-2-L 模型上，于单批解码场景中评估。

结果

在 GPT-2-L 上将单批解码延迟降低 3.16x，同时完美保持输出质量（已在 claim_checked 中核实）。该方法是 distribution-preserving (lossless) 的。

局限

面向小批量/端侧场景；在算术强度更高的大 batch 下收益可能缩小；在相对较小的模型上验证。

与领域的关系

将 SpecInfer 式的 tree speculation 与 speculative decoding 的递归（分阶段）应用相结合；是 Cascade Speculative Drafting 多层 drafting 的前身。
