# RouteLLM ## 一句话 - 基于人类偏好数据训练的 router 模型能针对每个查询动态选择强或弱的 LLM，在质量持平的前提下将成本削减 2 倍以上，并可跨模型对迁移。 ## 问题 - 学习一个 router，针对每个查询判断需要用弱模型还是强模型，从而优化成本与质量的权衡。 ## 洞见 - 人类偏好数据（两个模型答案中哪个更好）是训练 router 的强信号，而数据增强能让 router 更鲁棒、更易迁移。 ## 方法 - 一套训练框架利用经合成/增强技术扩充的人类偏好数据产出多种 router 架构；推理时 router 对查询打分，并将其分派给强或弱 LLM 以命中成本目标。 ## 设置 - 在广受认可的基准（如 MT-Bench、MMLU、GSM8K）上使用强/弱模型对进行评估；基线包括随机路由和已有 router。同时通过更换模型对来测试迁移能力。 ## 结果 - 在不损失响应质量的情况下将成本降低超过 2 倍；在更换强/弱模型时展现出很强的迁移学习能力。路由与奖励建模被明确区分开（已在 claim_checked 中核实）。 ## 挑战 - 获取足够的带标注路由数据、避免把路由与奖励建模混为一谈，以及确保在测试时更换强/弱模型对后 router 仍然有效。 ## 局限 - 聚焦于强 vs 弱的二元路由，而非多模型选择；性能取决于偏好数据的质量。

背景

部署时必须在能力强但昂贵的 LLM 与便宜但较弱的 LLM 之间取舍；固定单一选择会在简单查询上浪费金钱，或在困难查询上失败。

问题

学习一个 router，针对每个查询判断需要用弱模型还是强模型，从而优化成本与质量的权衡。

动机 / 分析

以往的路由工作缺乏有原则的训练数据，没有清晰地把路由与奖励/偏好建模区分开，也未证明在底层模型更换时的迁移能力。

关键洞见

人类偏好数据（两个模型答案中哪个更好）是训练 router 的强信号，而数据增强能让 router 更鲁棒、更易迁移。

挑战

获取足够的带标注路由数据、避免把路由与奖励建模混为一谈，以及确保在测试时更换强/弱模型对后 router 仍然有效。

方法

一套训练框架利用经合成/增强技术扩充的人类偏好数据产出多种 router 架构；推理时 router 对查询打分，并将其分派给强或弱 LLM 以命中成本目标。

实验设置

在广受认可的基准（如 MT-Bench、MMLU、GSM8K）上使用强/弱模型对进行评估；基线包括随机路由和已有 router。同时通过更换模型对来测试迁移能力。

结果

在不损失响应质量的情况下将成本降低超过 2 倍；在更换强/弱模型时展现出很强的迁移学习能力。路由与奖励建模被明确区分开（已在 claim_checked 中核实）。

局限

聚焦于强 vs 弱的二元路由，而非多模型选择；性能取决于偏好数据的质量。

与领域的关系

一个典范性的、基于偏好训练的二元 router；与多模型 router（Zooter、RouterDC、GraphRouter）形成对照，并在后续基准（RouterBench、LLMRouterBench）中常被用作基线。
