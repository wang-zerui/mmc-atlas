C. 角色专精编排

18 篇论文
由 manager 或流水线协调的 planner/worker/subagent 团队。

基础方法。 具名角色团队：MetaGPT (M011)、ChatDev (M050)、AutoGen (M010)、
CAMEL (M012)、AgentVerse (M051)；MacNet (M052) 借助逻辑斯蒂式的
"协作缩放律"扩展至 1000+ 智能体；Anthropic 的编排者-工作者系统
(M013) 报告称以约 15× 的 token 成本相较单个 Opus 4 提升 +90.2%。2026 SOTA。
出现两股相反的潮流。建设性一面：动态、自组织的编排——
AOrchestra（M151，ICML'26）从一个 4 元组即时生成子智能体（+16.28%）；
AdaptOrch (M152) 主张在 LLM 趋同的背景下拓扑结构支配模型选择；
HERA (M155) 与 AutoScientists (M154) 在无中心
规划者的情况下自组织团队。破坏性一面：2026 年的负面结果在此处冲击最猛——
Multi-Agent Teams Hold Experts Back（M153，ICML'26）表明团队即便被告知谁是专家，
仍无法匹敌其最佳专家（最多 −41.1%）；Ringelmann 效应
缩放律 (M156) 发现"30 个密集辩论的智能体产生的多样性不比
一个更多"；而多样性核算 (M157) 表明 2 个多样化智能体 ≈ 16 个同质
智能体。当结构是动态且异构时编排有帮助，
而当它强制同质共识时则适得其反。

本族论文

AOrchestra: Automating Sub-Agent Creation for Agentic Orchestration (Ruan et al. 2026)
AOrchestra 拥有一个中央 orchestrator，它从统一的四元组（Instruction、Context、Tools、Model）出发即时生成任务专属的 sub-agent，取代了手工预定义的静态角色。
M151ICML 2026
AdaptOrch: Task-Adaptive Multi-Agent Orchestration in the Era of LLM Performance Convergence (Yu 2026)
AdaptOrch 主张：随着各家提供商的 LLM 在基准上趋同，主导系统性能的是编排拓扑而非模型选择，并将每个任务的 DAG 动态路由到 parallel/sequential/hierarchical/hybrid 拓扑。
M152arXiv 2026 (preprint)
Multi-Agent Teams Hold Experts Back (Pappu et al. 2026)
一个负面结果：自组织的 LLM 团队始终无法达到其最佳个体专家的水平（在 ML 基准上损失高达 41.1%），即便被告知哪个 agent 是专家也是如此；瓶颈在于对专家的利用，而非对专家的识别。
M153ICML 2026
AutoScientists: Self-Organizing Agent Teams for Long-Running Scientific Experimentation (Gao, Fang, Zitnik 2026)
AutoScientists 是一个去中心化的多 agent 系统（没有中央规划者），其中的 agent 读取共享的实验状态，围绕有前景的假设自组织成团队，在投入算力之前对提案进行批判，并共享成功与失败。
M154arXiv 2026 (preprint)
HERA / Experience as a Compass: Multi-agent RAG with Evolving Orchestration (Li & Ramakrishnan 2026)
HERA 是一个分层的多 agent RAG 框架，它在无需梯度训练的情况下，联合演化查询专属的 agent 拓扑（reward-guided sampling）与角色专属的 prompt（Role-Aware Prompt Evolution），并报告出现了自组织形成紧凑高效用网络的现象。
M155arXiv 2026 (preprint)
The Ringelmann Effect in Multi-Agent LLM Systems: A Scaling Law for Effective Team Size (Bertalanic & Fortuna 2026)
推导出一条双参数的规模律 R(N)=N_eff/N，将 agent 配置划分为 hard-ceiling/sublinear/linear 三个区间，并从负面证明：30 个密集辩论的 agent 所增加的多样性不比 1 个更多，而一个 noise placebo 就能匹配 self-correction 的收益；只有异构团队才能突破天花板。
M156arXiv 2026 (preprint)
Understanding Agent Scaling in LLM-Based Multi-Agent Systems via Diversity (2026)
对增加 agent 为何会饱和的一种信息论解释：同质输出彼此相关，因此收益封顶于任务不确定性，而异构 agent 则增加了互补的信道；实证上 2 个多样化 agent 就能匹配或超过 16 个同质 agent。
M157arXiv 2026 (preprint)
Anthropic Multi-Agent Research System (2025)
一个 orchestrator-worker 式的研究系统：一个 Claude Opus 4 主导 agent 派生出多个并行的 Claude Sonnet 4 子 agent，各自独立检索并返回经过筛选的发现以供综合。
M013Anthropic engineering blog (2025)
MacNet / Scaling LLM Multi-Agent Collaboration (Qian et al. 2024)
MacNet 将 LLM agent 组织为有向无环图（DAG），可扩展到 1000+ 个协作 agent，并识别出一条呈 logistic 质量增长的“协作 scaling law”，其中 small-world/不规则拓扑表现最佳。
M052ICLR 2025
MetaGPT (Hong et al. 2023)
MetaGPT 将人类的标准化操作流程（SOP）编码进 prompt 序列，并采用流水线式的角色分工，以减少 LLM agent 团队中的级联幻觉（cascading hallucination）。
M011ICLR 2024 (oral)
ChatDev (Qian et al. 2023/ACL 2024)
ChatDev 是一个由聊天驱动的软件开发框架，角色专业化的 LLM agent（CEO/CTO/程序员/测试员）沿着一条“chat chain”协作，并借助 communicative dehallucination 端到端地构建软件。
M050ACL 2024
AgentVerse (Chen et al. 2023/ICLR 2024)
AgentVerse 通过一个由 recruit/decide/act/evaluate 四阶段构成的循环动态组建专家 agent 团队，实现了“整体大于部分之和”的系统，其表现优于单个 agent，并展现出涌现的社会行为。
M051ICLR 2024
DyLAN (Liu et al. 2023)
DyLAN 是一个 Dynamic LLM-Powered Agent Network，它用一个无监督的 Agent Importance Score 选出面向具体任务的团队，让 agent 动态协作，并停用贡献较弱的成员。
M053COLM 2024
Magentic-One (Fourney et al./Microsoft 2024)
Magentic-One 是一个通用型的 manager-worker 多 agent 系统：一个带有任务/进度台账（ledger）的 Orchestrator 指挥专业化的 Coder、Executor、FileSurfer 与 WebSurfer agent，在 GAIA、AssistantBench 与 WebArena 上取得有竞争力的结果。
M054arXiv 2024 (Microsoft; built on AutoGen)
OpenAI Swarm / Agents SDK (2024-2025)
OpenAI Swarm 是一个极简的编排库，构建在两个原语之上——routines（一个 agent = 指令 + 工具）和 handoffs（一个返回另一个 agent 的函数）——后来被生产级的 Agents SDK 所取代。
M055OpenAI (GitHub, Oct 2024) / Agents SDK (Mar 2025)
AutoGen (Wu et al. 2023)
AutoGen 是一个开源框架，由可定制、可对话的 agent 组成，它们通过相互发送消息来完成任务，并可混合使用 LLM、工具和人类输入。
M010arXiv 2023 (later COLM 2024 workshop / widely cited)
CAMEL (Li et al. 2023)
CAMEL 通过角色扮演与“inception prompting”，让两个交流型 agent 自主协作完成任务，同时生成用于研究 agent 行为的对话数据。
M012NeurIPS 2023
LangGraph vs CrewAI (framework references)
两种主流的生产级编排范式：LangGraph 将 agent 建模为显式的状态机图（带类型的共享状态、checkpointing、循环），而 CrewAI 将其建模为自上而下的基于角色的 crew（顺序式或分层式流程）。
M056Framework documentation (LangChain / CrewAI)
