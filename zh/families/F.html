F. 强弱协作提效

25 篇论文
弱模型承担大量工作,强模型验证或精修。

基础方法。 本族同时包含形式上无损的 speculative decoding 与有损语义上下文缩减。Speculative decoding（M005/M006）保持目标分布；EAGLE（M007）、SpecInfer（M081）、Staged（M082）和异构词表 TLI（M086）扩展了它。LLMLingua（M089/M090）、AgentDiet（M092）与 ACON（M093）则压缩 prompt 或 agent 轨迹。

轨迹压缩，2025–26。 按严格标准——后续模型调用必须读取由先前观测/动作/工具结果导出的缩减状态——现在可分五类：过滤（M323/M325/M333）、学习式持续状态（M322/M324/M330/M331/M340）、主动折叠（M327/M328/M334/M338）、可恢复外部证据（M326/M336），以及服务/前瞻控制（M337/M339）。RE-TRAC（M335，ICML'26）在轨迹之间写入类型化的证据、不确定性、失败和计划。Context-Folding、RE-TRAC 为 ICML'26；MEM1、AgentFold、IterResearch 与 domain-adaptive LightMem M340 为 ICLR'26。另一篇 2025 LightMem M329 已撤回。

不能把这些称为无损。 语义摘要和剪枝几乎总会丢信息。PAL-UI、U-Fold、MemexRL 因外部保留原始证据而“可恢复”，但检索仍可能失败。不同论文的活跃上下文、累计 token、FLOPs、轮数、墙钟时间和成功率也不可直接互换。ReSum 被 desk reject，R_FOLD 尚未录用，Focus M332 只有 N=5。相比之下，Saguaro（M190）与 BASTION（M192）等 2026 speculative decoding 工作仍有分布保持的结构性保证；VSD/PARSE/TreeFlash 仍是近似权衡。

本族论文

AgentDiet (Xiao et al. 2025)
一种推理时的轨迹缩减方法，自动从软件 agent 不断增长的上下文中剪除冗余、无用与过期的信息，在不损害性能的前提下削减 input-token 成本。
M092FSE 2026
ACON: Agent Context Optimization (Kang et al. 2025)
从失败分析中优化自然语言压缩准则，再将 LLM compressor 蒸馏进一个小模型，使弱模型为强长程 agent 精简上下文。
M093ICLR 2026 Lifelong Agent Workshop
MEM1: Learning to Synergize Memory and Reasoning for Efficient Long-Horizon Agents (2025)
每轮把推理结果写回一个紧凑的共享记忆状态，以此替代无界增长的原始历史。
M322ICLR 2026 Poster
ReSum: Unlocking Long-Horizon Search Intelligence via Context Summarization (2025)
周期性地把增长中的搜索轨迹总结为紧凑推理状态，并训练策略跨越总结边界继续工作。
M324arXiv preprint; ICLR 2026 desk reject
FocusAgent: Context-Aware Active Observation Pruning for GUI Agents (2025)
用轻量语言模型检索器移除无关的可访问性树观测，再交给主 GUI agent。
M325ICLR 2026 Lifelong Agent Workshop
Scaling Long-Horizon LLM Agent via Context-Folding (2025)
把子任务分支到独立上下文中执行，完成后只把紧凑的返回摘要折叠回父轨迹。
M327ICML 2026
AgentFold: Long-Horizon Web Agents with Proactive Context Folding (2025)
主动在细粒度局部折叠与深度整合之间选择，而不是等触及上下文上限后再做一次通用总结。
M328ICLR 2026 Poster
LightMem: Lightweight and Efficient Memory-Augmented Generation (2025)
压缩新经验、按主题整合短期记忆，并把长期记忆更新移到关键执行路径之外。
M329ICLR 2026 withdrawn submission
IterResearch: Rethinking Long-Horizon Agents with Interaction Scaling (2025)
以持续演化的报告作为跨轮紧凑记忆，并周期性重建工作状态，而不是重放全部研究交互。
M330ICLR 2026 Poster
AgeMem: Agentic Memory for LLM Agents (2026)
把存储、检索、更新、总结和丢弃都作为可学习的记忆工具，让 agent 主动压缩自身轨迹状态。
M331arXiv preprint
Focus: Active Context Compression for LLM Agents (2026)
让 agent 在执行中主动整合已学事实并剪除原始历史，而不是依赖固定的溢出压缩器。
M332arXiv preprint
SWE-Pruner: Goal-Guided Context Pruning for Software Engineering Agents (2026)
使用由任务目标条件化的 0.6B 神经 skimmer，在昂贵推理调用前剪除软件 agent 上下文。
M333arXiv preprint
U-Fold: Intent-Aware Context Folding for Long-Horizon Agents (2026)
维护意图感知的演化对话摘要和紧凑动作/工具日志，同时把原始历史留在外部存储。
M334arXiv preprint
RE-TRAC: REcursive TRAjectory Compression for Deep Search Agents (2026)
每条轨迹后写出包含证据、不确定性、失败和计划的结构化状态；下一轮读取该状态而非原始轨迹。
M335ICML 2026
MemexRL: Learning to Build Recoverable Memory for Long-Horizon Agents (2026)
学习写入紧凑结构化摘要与稳定索引，同时把完整原始交互存入外部数据库供选择性解引用。
M336arXiv preprint
Parallel Context Compaction for Long-Context Agents (2026)
并行压缩独立上下文块，把语义总结变成可预测的服务原语，而不是串行停顿。
M337arXiv preprint
R_FOLD: Read-and-Fold Context Management for Web Agents (2026)
结合步内观测过滤/蒸馏与跨步续行摘要，让网页 agent 在固定上下文预算内运行。
M338ACL ARR May 2026 submission; not accepted
AgentSwing: Lookahead Routing over Context-Compaction Strategies (2026)
并行运行 Keep-Last-N、Summary、Discard-All 三种压缩分支，再用 K 轮前瞻选择未来轨迹最好的分支。
M339arXiv preprint
A Lightweight, Domain-Adaptive Memory System for LLM Agents (LightMem, 2026)
把长交互历史缩减为提取—整合—检索流水线，其分层记忆可跨个性化与代码仓库任务迁移。
M340ICLR 2026 Poster
Less is More: Consistent GUI Agents with Context-Aware Simplification (SimpAgent, 2025)
结合 GUI 元素遮蔽与一致性引导的历史压缩，让后续动作读取更小的观测和轨迹。
M323arXiv preprint
PAL-UI: Progressive Agent Learning for Long-Horizon GUI Interaction (2025)
维护观测与动作的双层摘要，同时把原始截图保留在外部存储中以便后续回取。
M326arXiv preprint
LLMLingua-2 (Pan et al. 2024)
将 GPT-4 的压缩知识蒸馏进一个小型双向 encoder，以 token classification 的方式进行 prompt 压缩，比 LLMLingua 更快、更忠实。
M090ACL 2024 (Findings)
Big Little Decoder (Kim et al. 2023)
一个小模型自回归地解码，而大模型仅被偶尔调用，以非自回归的方式仅精修不确定的预测，通过学习到的 fallback 与 rollback 策略实现。
M008NeurIPS 2023
LLMLingua (Jiang et al. 2023)
一个小型对齐的 LM 在大模型读取之前对 prompt 进行 coarse-to-fine 压缩，将上下文缩减这一任务委托给弱模型以削减成本。
M089EMNLP 2023
Orca (Mukherjee et al. 2023)
把 distillation 作为 strong-weak 迁移：一个 13B 模型模仿 GPT-4 的逐步解释轨迹（而不仅是输出风格），这是一种训练时的耦合，之后可独立运行。
M091arXiv 2023 (Microsoft Research)
