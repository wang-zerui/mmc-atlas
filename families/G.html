G. Communication & Memory Protocols

9 papers
Standards for how instances pass messages, share state, expose tools.

Base. MCP (M020) standardizes agent↔tool; A2A (M095) agent↔agent; a survey
(M096) lays out MCP/ACP/A2A/ANP; memory substrates MemGPT (M098) and A-MEM
(M099). 2026 SOTA. Beyond-Message-Passing (M097) formalizes a 3-layer
(comm/syntax/semantics) taxonomy and shows current protocols under-support
*semantic* alignment (clarification/repair pushed into prompts as technical debt);
Web-of-Agents (M212, ICML'26 position) argues minimal shared standards are needed
to avoid fragmentation; ProtocolBench (M211) isolates protocol effects (completion
time varies up to 36.5%) — though its ICLR'26 status is under-review, not
confirmed; Mesh Memory (M213) proposes cross-session cognitive sharing. Protocols
are maturing syntactically; the open gap is semantic.

Papers in this family

Beyond Message Passing: A Semantic View of Agent Communication Protocols (Yuan et al. 2026)
This paper organizes agent communication protocols into a three-layer taxonomy (communication, syntactic, semantic) inspired by Shannon-Weaver and Gricean pragmatics, and shows that current protocols are mature on transport/syntax but under-support semantic alignment such as clarification and repair.
M097arXiv 2026 (preprint)
ProtocolBench / ProtocolRouter (Du, Zhu et al. 2025/2026)
ProtocolBench isolates the effect of the agent communication protocol by holding LLM, prompts, and hardware constant while varying the protocol (A2A/ACP/ANP/Agora), and ProtocolRouter is a learnable per-module router that selects protocols from requirement and runtime signals.
M211ICML 2026 (accepted per arXiv v3, Jun 2026)
Position: Collaborative Agentic AI Needs Interoperability Across Ecosystems (Sharma et al. 2026)
This ICML 2026 position paper argues that collaborative agentic AI needs minimal shared standards to avoid fragmenting into incompatible ecosystems, and proposes 'Web of Agents' as a minimal architectural foundation built from four components.
M212ICML 2026 (position paper)
Mesh Memory Protocol (Xu 2026)
The Mesh Memory Protocol (MMP) is a proposed semantic-infrastructure layer that lets teams of LLM agents share, evaluate, and combine each other's cognitive state across sessions, built from four composable primitives (CAT7, SVAF, inter-agent lineage, remix).
M213arXiv 2026 (v0.2.3 specification, CC BY 4.0)
A2A (Agent2Agent) Protocol (Google 2025)
A2A is an open peer-to-peer protocol that lets AI agents built on different frameworks and by different vendors discover one another, delegate tasks, and share results without exposing their internal memory, tools, or logic.
M095Open standard (Google, Apr 2025); donated to the Linux Foundation Jun 2025
Survey of Agent Interoperability Protocols (Ehtesham et al. 2025)
A survey that compares four emerging agent interoperability protocols - MCP, ACP, A2A, and ANP - along interaction, discovery, communication, and security, and proposes a phased adoption roadmap for building interoperable LLM-agent ecosystems.
M096arXiv 2025 (preprint)
A-MEM: Agentic Memory for LLM Agents (Xu et al. 2025)
A-MEM is a Zettelkasten-inspired agentic memory in which each memory is an atomic note with LLM-generated keywords, tags, and contextual description, dynamically linked to related notes and allowed to evolve as new memories arrive - moving beyond static RAG retrieval.
M099NeurIPS 2025
Model Context Protocol (Anthropic 2024)
MCP is an open JSON-RPC standard, likened to a 'USB-C port for AI', that connects AI applications to external data sources, tools, and workflows through a common client-server interface.
M020Open standard (Anthropic, Nov 2024); donated to the Agentic AI Foundation (AAIF) Dec 2025
MemGPT (Packer et al. 2023)
MemGPT applies operating-system memory-hierarchy ideas to LLMs, letting the model self-manage tiered memory - a fixed main context (like RAM) plus external storage (like disk) - via function calls, so it behaves as if it had a far larger context window.
M098arXiv 2023 (later COLM 2024); now the Letta framework
