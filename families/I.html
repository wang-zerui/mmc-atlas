I. Infrastructure

23 papers
Serving, runtime, scheduling, and deployment infrastructure for multi-model collaboration.

Scope. Infrastructure enters this survey only when it directly exploits a collaboration graph, multiple model calls, program state, or cross-agent exchange; generic multi-model hosting and ordinary single-model serving stay adjacent. The base request engine remains vLLM/PagedAttention (M109), SGLang/RadixAttention (M110), Parrot (M111), and DistServe (M112), but the closer 2025 work is workflow-aware: TokenCake (M376), KVFlow (M377), Continuum (M375), Astraea (M388), and Sherlock (M387) optimize graph criticality, tool/I/O stalls, end-to-end JCT, verifier placement, speculation, or rollback.

Agent-native runtime, 2026. Agentix (M371, NSDI'26) schedules by program progress; Helium (M374) treats workflows as query plans; ThunderAgent (M378) co-manages KV and tool environments; Scepsy (M381) plans GPU allocations for arbitrary multi-LLM pipelines; Pythia (M383) exports topology semantics; HeraSys (M223) merges nodes across workflows. Murakkab (M386, OSDI'26) decouples declarative collaborative workflow logic from model/hardware placement and jointly optimizes them under SLOs. LLM-as-Scheduler (M373, ACL'26) links query routing to whole-workflow selection. TokenDance (M382), RelayCaching (M222), PrefillShare (M221), and Stateful Inference (M379) exploit round-, cross-agent-, cross-model-, or cross-turn state reuse.

Limits. Per-agent resource isolation remains thin. M226 is the critical counterexample: reuse that accelerates generators can break judges by weakening cross-candidate attention.

Papers in this family

PrefillShare (Woo et al. 2026)
PrefillShare factorizes a model into prefill and decode modules, freezes the prefill module, and fine-tunes only the decode module, so multiple task-specific models can share one prefill stage and its KV cache in a vLLM-based disaggregated serving setup.
M221arXiv 2026 (cs.LG/cs.DC)
RelayCaching (Geng et al. 2026)
RelayCaching is a training-free method that recycles earlier agents' decoding-phase KV caches into later agents' prefill, selectively recomputing only the sparse positions that deviate, to cut cascading context-redundancy overhead in LLM collaboration pipelines.
M222arXiv 2026 (cs.LG/cs.AI)
HeraSys (Li et al. 2026)
HeraSys is a cross-workflow serving system for concurrent multi-tenant agentic workflows that eliminates inter-workflow redundancy via structural node merging/reuse, plus load-aware joint scheduling and resource skewing with adaptive batching.
M223ICML 2026
Software-Defined Agentic Serving (Agarwal et al. 2026)
An SDN-inspired framework arguing that multi-agent LLM serving systems should be programmable and system-aware, dynamically controlling communication attributes based on runtime state instead of statically encoded parameters.
M224arXiv 2026 (cs.DC/cs.MA)
Agentix: An Efficient Serving Engine for LLM Agents as General Programs (Luo et al. 2026)
Treats agent programs as first-class serving units and schedules calls by program progress, reducing call- and program-level head-of-line blocking.
M371NSDI 2026
LLM-as-Scheduler: Agentic Workflow Dynamic Scheduling (Xiang et al. 2026)
Chooses a workflow per query with a lightweight output gate followed by an LLM scheduler, connecting routing decisions to runtime cost and latency.
M373ACL 2026
Helium: Efficient LLM Serving for Agentic Workflows (Wadlom et al. 2026)
Models agent workflows as query plans whose LLM operators can be jointly scheduled with proactive prompt, KV-state, and cross-workflow caching.
M374arXiv 2026
ThunderAgent: A Program-Aware Agentic Inference System (Kang et al. 2026)
Represents workflows as LLM Programs and jointly manages KV, system state, and external tool resources such as disk and ports.
M378arXiv 2026
Stateful Inference for Low-Latency Multi-Agent Tool Calling (Norgren 2026)
Keeps per-session KV alive across turns and combines radix interleaving with lookup decoding so each tool-call turn ingests only new tokens.
M379arXiv 2026
Scepsy: Serving Agentic Workflows Using Aggregate LLM Pipelines (Wagenländer et al. 2026)
Profiles stable aggregate model-time shares to predict and place branching, fan-out, or recurrent multi-LLM workflows on oversubscribed GPU clusters.
M381arXiv 2026
TokenDance: Scaling Multi-Agent LLM Serving via Collective KV Cache Sharing (Bian et al. 2026)
Exploits synchronized all-gather rounds to reuse shared KV once collectively and store sibling agent caches as sparse diffs.
M382arXiv 2026
Pythia: Exploiting Workflow Predictability for Agent-Native LLM Serving (Yu et al. 2026)
Captures multi-agent topology semantics at the serving layer so cache, contention, queueing, and scaling policies can exploit workflow predictability.
M383arXiv 2026
Murakkab: Resource-Efficient Agentic Workflow Orchestration in Cloud Platforms (Chaudhry et al. 2026)
Decouples collaborative workflow logic from model and hardware choices, then jointly optimizes orchestration, placement, and runtime reconfiguration under SLOs.
M386OSDI 2026
Continuum: Multi-Turn LLM Agent Scheduling with KV Cache Time-to-Live (Li et al. 2025)
Pins multi-turn KV through short tool pauses using reload- and queue-aware TTLs, then evicts it automatically when retention stops paying off.
M375arXiv 2025 (rev. 2026)
TokenCake: A KV-Cache-centric Serving Framework for Multi-Agent Applications (Bian et al. 2025)
Co-optimizes tool-stall offload/prefetch and graph-aware memory partitioning so critical-path agents retain KV under spatial and temporal contention.
M376arXiv 2025
KVFlow: Efficient Prefix Caching for LLM-Based Multi-Agent Workflows (Pan et al. 2025)
Uses an Agent Step Graph to evict by steps-to-execution and prefetch the next agents' KV instead of applying workflow-blind LRU.
M377arXiv 2025
Sherlock: Reliable and Efficient Agentic Workflow Execution (Ro et al. 2025)
Finds error-prone workflow nodes, attaches cost-optimal verifiers, speculates past checks, and rolls back to the last verified state on failure.
M387arXiv 2025
Astraea: A State-Aware Scheduling Engine for LLM-Powered Agents (Ni et al. 2025)
Optimizes whole-workflow completion with history- and prediction-aware hierarchical scheduling plus adaptive KV handling during API waits.
M388arXiv 2025
Faster Cascades via Speculative Decoding (Narasimhan et al. 2024)
Combines cascade deferral with speculative verification into 'speculative cascades', implementing the deferral rule via speculative execution for better cost-quality than either technique alone.
M021arXiv 2024 (cs.CL)
Parrot / Semantic Variable (Lin et al. 2024)
Parrot exposes application-level dataflow to the LLM service via 'Semantic Variables' that annotate request I/O, enabling data-flow analysis across correlated requests for DAG-aware scheduling, cross-request batching, and prefix sharing.
M111USENIX OSDI 2024
DistServe (Zhong et al. 2024)
DistServe disaggregates the prefill and decode phases onto different GPUs to remove phase interference and independently optimize TTFT versus TPOT, serving far more requests within latency SLOs.
M112USENIX OSDI 2024
vLLM / PagedAttention (Kwon et al. 2023)
PagedAttention manages the KV cache like OS virtual-memory paging, giving near-zero fragmentation and ref-counted block sharing within and across requests, which lets vLLM batch more requests and raise serving throughput 2-4x.
M109SOSP 2023 (Best Paper)
SGLang / RadixAttention (Zheng et al. 2024)
SGLang pairs a frontend language for structured LLM programs with a runtime whose RadixAttention automatically reuses KV cache across generation calls via an LRU radix tree, reaching up to 6.4x higher throughput.
M110arXiv 2023 (NeurIPS 2024)
